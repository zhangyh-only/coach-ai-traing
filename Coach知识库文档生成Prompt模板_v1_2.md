# Coach 知识库文档生成 Prompt 模板

> **版本**：v1.2
> **更新日期**：2026-06-17
> **配套架构**：《Coach 陪练知识库·架构设计文档 v1.4》
> **适用范围**：本期 KB1–KB5

**用法**：处理一份原始资料时 → 先做 KB1-KB5 覆盖分析 → 判断它能更新哪些知识类型 → 对每个需要生成/更新的 KB 取对应 prompt → 把"通用说明前缀"拼到开头 → 填占位符（`{{分类}}`、`{{名称}}`、`{{日期}}`、`{{数据来源}}`）→ 末尾粘贴原始资料 → 交给大模型。单次 prompt 仍建议一次只生成一篇，但同一课程可以依内容顺序更新多个 KB 文档。

**占位符约定**：`{{分类}}` = 库内主题分类（如"正价""奥莱""通用""画像""流程""方法"）；`{{名称}}` = 文档主体名（产品名 / 阶段名 / 方法论名 / 画像名等）；`{{日期}}` = 今天；`{{数据来源}}` = 源文件名或资料类型。

---

## 一、通用说明前缀（所有 prompt 共用，拼在每个 prompt 开头）

```
你是 Coach（蔻驰）AI 陪练系统的知识库文档编写专家。你需要将我提供的原始资料转换为标准格式的知识库 MD 文档。

【知识库体系背景】
这个知识库服务于 AI 陪练训练系统：AI 扮演一位 GenZ（Z 世代）轻奢顾客，与导购学员进行模拟对话训练。
角色称谓固定为"导购（学员）—顾客"（用"顾客"，不用"客户"）。
Coach 是轻奢/时尚零售品牌，品牌理念为 Expressive Luxury，销售核心叙事是"经典长期主义 / 质感投资 / 正品保障"。沟通风格贴近 GenZ、平等、带朋友感，禁止传统奢侈品"端着"的官方客服腔。

知识库文档会在以下环节被引用（本期重点为前两项）：
1. 陪练对话【本期主线】：AI 扮演顾客时，引用画像/产品 FAQ/服务细节/异议表达，决定"她怎么想、怎么开口、被什么话术推开"
2. 对话评估【本期同步沉淀】：评判学员表现时，引用销售技巧定义、沟通禁忌、产品知识点、服务边界、画像理解和场景动作作为判断依据
3. 对话采集【后续】：识别产品/渠道/画像并匹配补全
4. 变量生成【后续】：生成陪练场景配置变量

【文档格式要求】
1. 标准 Markdown 格式
2. 开头必须包含元信息区块（文档编号、所属知识库、版本、更新日期、数据来源）
3. 必须包含"在陪练系统中的应用"章节，说明本文档在陪练对话/对话评估中如何被引用（采集/生成如适用可一并写，否则可省）
4. 必须包含"关联文档"章节
5. 内容完整、自洽，不依赖其他文档即可独立理解
6. KB 文档是"被检索、被提炼的参考材料"，不是"被整段复制的内容块"——为大模型检索理解服务
7. 资料里没有的字段标注"【待客户补充】"，绝不编造

【文档命名规范】
- 格式：KB[库编号]-[分类]-[简短标题].md（KB 大写、无序号、三段用 - 连接）
- 示例：KB2-正价-Tabby经典款.md、KB4-画像-质感自用型.md、KB3-流程-迎宾破冰阶段详解.md、KB3-方法-TA Card销售技巧.md

【数据来源填写规范】
- 源为企业培训资料：直接用原始文件名，多文件顿号分隔
- 源为企业直接提供：Coach 提供·资料类型（YYYY年Qx）
- 暂无源文件：填【待客户补充源文件】

【资料使用优先级】
1. Coach 课程资料 / 课程分析中间稿是 KB 正文的事实主来源，用来写品牌、产品、材质、服务、退换、方法论、话术、沟通原则等可被评估和对话引用的内容。
2. 新场景要求、业务老师整理资料、China Persona、角色图用于确定陪练场景、顾客分类、画像方向、渠道属性和生成优先级；它们不能替代课程资料来编写产品参数、服务规则或方法论定义。
3. 如果课程资料与业务老师资料冲突，正文优先保留课程资料口径，并在"来源融合记录"或"待确认事项"中列出冲突，等待业务确认。
```

## 六、顾客画像类文档生成 Prompt（KB4）★ 本期核心

```
{{通用说明前缀}}

【本次任务】请将原始资料转换为一份**顾客画像知识库文档**。这是本期最重要的文档类型，直接决定 AI 顾客"演得像不像"。
【文档名称】：KB4-画像-{{画像名称}}

【特别要求】
1. 本文档既要让 AI 知道"这个顾客是谁"，更要让 AI 知道"她会怎么开口、被什么话术推开、心里有哪些没说出口的焦虑"——所以必须写满"角色扮演行为指引"字段。
2. 异议一律写成"关切主题 + 心理/行为倾向"的口语化短句（10-25 字），**绝不写完整问句台词**（完整台词会让 AI 机械复读，失去自然感）。
   - 反例（不要）：写出"你们这个比线上贵这么多，凭什么？"这类整句
   - 正例（要）：对线下溢价敏感、希望听到一个能说服自己的理由
3. 资料里没有的字段标"【待客户补充】"，不编造。
4. 不需要等待专门的 Persona 或画像资料。课程中的购买动机、适合人群、使用场景、顾客异议、没说出口的担心、社媒偏好、送礼/纪念/悦己心理、服务焦虑、被哪类话术推开，均可作为 KB4 事实来源。

【多源融合规则】
1. KB4 主分类固定为 4 类行为类型：质感自用型 / 性价比入门型 / 礼品选购型 / 都市悦己型。
2. China Persona A/B、角色图 A/B 只作为"渠道画像补充来源"，用于补年龄、城市、职业、收入、渠道偏好、客单价、退货倾向、社交依赖等字段；不能把 A/B 直接当成 KB4 主分类。
3. 如果原始资料可明确映射到某个行为类型，要在"来源融合记录"说明依据；如果只能判断渠道、不能判断行为类型，必须标注"【待人工确认】"，不要强行归并。
4. 同一行为类型在不同渠道表现不同，可以在"渠道属性"中分写线下正价 / 奥莱 / 私域差异，但不要拆成多个互斥画像，除非后续业务明确要求独立检索。
5. 如果课程线索无法稳妥映射到单一行为类型，优先更新 `KB4-异议-*` 或 `KB4-潜台词-*` 汇总类文档；不要为了凑画像而强行归类。
```

## 八、文档更新 Prompt 与使用流程

### 8.1 何时更新 vs 重建
- **更新**：客户补充了新资料、画像异议策略微调、某条服务规则变化 → 在已有文档基础上更新。
- **重建**：新资料与现有结构差异极大 → 回到对应生成 prompt 重新生成。

### 8.2 文档更新 Prompt

```
你是 Coach AI 陪练系统的知识库文档维护专家。我将提供一份已有知识库文档和新的补充/修订资料，请对比两者，将新资料中有价值的内容提炼后更新到现有文档中。

【更新原则】
1. 以新资料为信息来源，判断哪些需要新增/修改/替换；未涉及的内容保持原文不变
2. 新资料与原文矛盾时以新资料为准
3. 保持原文档结构和格式规范不变；新资料涉及原结构未覆盖的内容，在最合适章节补充
4. 版本号递增（v1.0→v1.1），更新日期改为今天，数据来源字段补充本次新资料来源
5. 输出完整的更新后文档，不要只输出变动部分
6. 画像类文档更新时，"角色扮演行为指引"和"异议（关切主题）"字段尤其要保持不写台词的原则

【当前文档如下】：
{{粘贴当前完整文档内容}}

---

【新补充/修订资料如下】：
{{粘贴新的原始资料}}
```

### 8.3 使用流程要点
1. 一份资料 → 先做 KB1-KB5 覆盖分析 → 判断需要更新哪些知识类型 → 逐篇选对应 prompt → 拼通用前缀 → 填占位符 → 末尾粘资料 → 交大模型。
2. **单次 prompt 一次只生成一篇**，不要让大模型在一个 prompt 中同时产出多篇（质量下降）；但同一课程可依内容顺序更新多个 KB 文档。
3. 检查输出：未涉及内容是否被误改、版本号/日期是否更新、异议是否误写成完整台词、是否有该填"【待客户补充】"却被编造的字段。
4. 保存为规范文件名 → 上传平台并打 `KB 编号` + `分类名称` 标签。

## 九、v1.5 补充：按库重组 + KB4 索引类 + KB5 功能分层

**9.1 按库重组（废止"一课一主 KB"）**：一门课贡献几个 KB 层就更新几个，不要为"一课一主"而压制 KB4/KB5 等次级库。跨课的同检索单元（如多门课都讲皮革材质、都涉及某画像异议）合并进同一篇 KB，并在数据来源列出所有来源课程。

**9.2 KB4 七篇 = 4 画像 + 3 索引**：
- 画像类（质感自用型/性价比入门型/礼品选购型/都市悦己型）：用第六节画像 prompt，必填"角色扮演行为指引"，异议写"关切主题+潜台词"。基础人设无课程依据时用 China Persona 量化属性补，仍缺标【待客户补充】；A/B 画像→4 行为类型的映射在"来源融合记录"说明，不确定标【待人工确认】。
- 索引类（异议-类型分类总表 / 异议-阶段分布索引 / 潜台词与决策焦虑索引）：去掉角色扮演指引，核心是横扫多门课 KB4 候选 + 业务老师异议框架做的归类表。

**9.3 KB5 功能分层四件套（不再按内容主题堆叠）**：
- KB5-设计-评估体系设计原则（能力观 + 禁止指标 + 红线 + 总分100）
- KB5-维度-核心评估维度参考库（Coach 定稿四维度：专业性30/同理心25/沟通技巧25/流程意识20，每维度=能力本质+判断逻辑+课程检查点+三档）
- KB5-评分-评分规范（权重 + 优秀/良好/待提升档位 + 打分方法）
- KB5-场景-场景类型×维度匹配（渠道×6阶段×客户类型→维度权重偏移）
> KB5 记录"能力判断逻辑"，与评估变量格式版本解耦；课程要求→可观察检查点挂到对应维度；权重/阈值待历史样本校准。
