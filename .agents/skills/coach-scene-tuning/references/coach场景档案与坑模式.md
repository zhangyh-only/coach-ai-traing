# Coach 坑模式库 + 场景档案 + 迭代史沉淀

这是本 skill 的"深度经验层"。诊断时拿新现象**对照坑模式比对**——coach 历史踩过的坑，大多能直接命中根因和落点。落点一律用业务侧名称（映射见 `coach提示词结构与变量映射.md`）。

## 第一部分：Coach 七大坑模式（C1–C7）

每条给：怎么发现（信号）→ 典型根因 → 落点与改法 → 来源。这些是 coach 角色扮演链反复出现的、比通用方法论更具体的坑。

### C1. 状态连续性失忆：把已发生的当没发生、从头再来

- **信号**：Elena 重复已做过的动作（已试背还问"我能上身试一下吗"）、重复已申请且被同意的动作（SA 已说"可以试"，下一轮又"我得自己塞东西试试")、重复已答过的问题、对话中段莫名蹦回"我先随便看看"的开场姿态。
- **根因**：**直调模型每轮都喂了全量历史、它"看得见"自己说过什么，但提示词没把"每轮开口前先认一遍这局已发生什么"写成硬约束**——这不是模型真失忆，是缺约束。直调比 Agent 更依赖这条显式写死（Agent 还有些隐式上下文兜底，直调全靠提示词）。
- **落点与改法**：【角色扮演链 → ★铁律 → 第1条·认历史不重来】写死"每次开口前在心里过一遍：聊到哪件事了、包摸过/背过没、哪些问题已问过他也答过、旧包提过没"，且**旁白里演过的动作也算'已发生'**（见 C2）。配套【铁律 → 第2条·没接住一句≠进度清零】给"无效输入轮"一条不重置的出口（拎回上一问/请重说，绝不退回开场）。
- **来源**：coach v8.5「状态连续性约束」草案立项动因（业务老师 11 条反馈）；2026-06-25 测试 loop 3→5（已同意试塞还重申）、loop 4→6（旁白试背了还问能上身试）复发。
- **注意**：这条是 coach 的"必须显式写死"特例——**不要按通用铁律1"改写法别加规则"去删它**。记忆问题加约束，风格问题才改写法。

### C2. 旁白机制坑：动作不算已发生 / 旁白过载 / 写进心理 / 未完成态

- **信号**：①旁白演过的动作没被下一轮当"已发生"（loop4 旁白"试着把肩带往肩上挂"，loop6 又问"能上身试一下吗"）；②几乎每轮都带括号动作（实测 13 轮 12 轮带括号 = 92%），违反"多数回合不加括号";③括号里同质复读（"拿起 Brooklyn 28 + 指尖摩挲/按了按"反复出现 5+ 次）；④括号里写心理（"心动""犹豫"）。
- **根因**：①旁白与口语在模型眼里是"两套独立轨道"，生成下一轮口语时没把旁白动作算进历史；②"旁白要克制"写成了软建议、没写成硬上限；③旁白措辞用了**未完成态**（"试着把肩带往肩上挂"读起来像"正要挂"，模型据此认为还没试背）。
- **落点与改法**：【角色扮演链 → 旁白机制】钉死三条——(a) 把"整场也就两三个关键时刻值得加旁白（第一次被吸引、第一次上手、最后递包）"从建议**升为上限**，明确"多数回合不加括号"；(b) "旁白里演过的动作=已发生、不重做"（与铁律1呼应）；(c) **动作要么不写、要写就写完成态**（"她把包背上了肩"而不是"她试着把肩带往肩上挂"），别给模型"动作没做完所以还能再申请"的空子；(d) 括号只写看得见的动作 + 点明在看哪只包，绝不写任何心理/情绪。
- **来源**：2026-06-25 测试 loop4/6（未完成态导致重复申请）+ 全程旁白过载 92%；coach v8.5 第二批「旁白机制」重构。

### C3. 顾客越界：查知识库 / 报参数 / 复述卖点话术

- **信号**：Elena 嘴里冒出只可能来自资料的话（"每只独一无二""头层牛皮""植鞣工艺"）、报出精确尺寸容量、或一本正经核对导购参数对不对；百炼观测里出现 `search_knowledgebase` span、单轮延迟/ token 异常高。
- **根因**：早期把知识库挂给了角色扮演链、让 AI 顾客自查 KB，导致它**站到了"产品专家"视角**、复述卖点、被产品资料污染（真实 trace：34s / 1.8 万 token）。
- **落点与改法**：架构上**角色扮演链解绑知识库、关 `search_knowledgebase` 工具**（ADR 0004，属平台层、由技术执行）；提示词侧【角色扮演链 → ★铁律 → 第5条·产品事实边界】写死"顾客是**外部视角**——产品事实事先不知道、不查库、不报参数、不自己编材质学名/数字，想知道就用大白话问导购、等他答；识货靠常识追问、但不负责核对他参数报得准不准（事实核查归评估链）"。
- **来源**：coach ADR 0004 + v8.5 第三批；这是 coach 区别于"知识考核类"场景的根本定性——Elena 是外部顾客、不是考官。

### C4. 看得见/常识可估的，还张口问导购

- **信号**：①已（旁白）打开/翻看过包内里，却又问"里面有没有隔层/拉链袋"（看得见的结构）；②**见到实物、上手了，还问"这大小装得下 A4 纸或大点的保温杯吗"**（大小是眼睛+常识就能估的体感，不是"精确容量"）。
- **根因**：提示词早期把"尺寸容量"一刀切成"顾客不预先知道、可以问"，但**没区分"精确容量参数"（确实要问）和"目测体感"（眼睛+常识就能估，不该问）**；"看得见的别问"那条只覆盖了"已打开的结构"，没覆盖"目测大小够不够装"。这是 v9 也没补到的缺口。
- **落点与改法**：【角色扮演链 → ★铁律 → 第5条·产品事实边界】在"看得见的别问'有没有'"基础上**增一类**——"大小够不够塞下 A4/文件夹/水杯/雨伞这类**眼睛瞄一下 + 常识就能估**的，自己说体感判断（'这大小日常通勤够用了''看着塞个文件够呛'），别开口问导购'装得下吗'。真要问，只问看不见、估不出的（精确能装多少、皮料学名、能用几年、保不保修）"。
- **来源**：2026-06-25 测试 loop4（"装得下 A4/保温杯吗"）+ loop9（"塞个文件夹、雨伞应该没问题吧"）——用户列为问题①，且与 C5 叠加。

### C5. 同一担忧换个表皮反复抛（异议复读）

- **信号**：同一个底层关切，换了说法/换了物件名又问一遍。典型：容量类——loop4"装得下 A4/保温杯吗"→ loop9"塞个文件夹、雨伞行吗"（都是"够不够装我通勤用品"）；版型类——loop2"软皮背久了没型"（SA 已答）→ loop5"东西多了会不会撑变形"；Bella 奥莱打理类——先问材质名/绒面清洁/护理服务，导购答过后又隔几轮回到材质学名和好不好打理。
- **根因**：①提示词的"别重复抛同一担忧"约束不够硬、没点明"换个物件/换个说法 = 还是同一个担忧"；②与 C7、C4 联动——没东西可推进时，模型靠"换皮重问"来填满回合。
- **落点与改法**：【角色扮演链 → ★铁律 → 第1条】末段强化"**同一个底层担忧（够不够装、会不会变形、耐不耐刮），导购接住过一次，就别换个物件名/换个说法再抛一遍**——A4 和文件夹是同一个'够不够装'，软皮没型和撑变形是同一个'会不会塌'；他答得不满意你可以接着这个点往下追，但不是从头再抛一遍"。注意 C4 修好后会顺带拦掉容量类换皮。**警惕反作用**：硬禁复读后模型可能"为了说新的"去翻新顾虑，需与 C7"答到位就推进/收尾"一起改、一起验。
- **来源**：2026-06-25 测试，用户列为问题②④。

### C6. 随机性格层穿不透：4 种性子跑出同一份行为指纹

- **信号**：跑多局、抽到不同性格卡（A 理性/B 比价/C 冷淡/D 心动），但开场都是"随便看看"、每局都奔"什么皮/会不会塌/划痕/肩带"去问、单条都 150–220 字爱解释——性子没体现差异。
- **根因**：**提示词过度圈定把随机性格压平了**——5 阶段写死"标准问题清单"（每局都照着问材质）、逐字示范句变台词、固定画像引力把所有性子拉向同一行为；次因才是性格卡 content 写得太简单/同质。这是**两层都要改**的典型（L1 提示词 + L2 性格卡咬合）。
- **落点与改法**：①【角色扮演链 → 这一局的你/对话指导】把性子提为**总开关并钉死辖域覆盖全文**、明确它**优先于**正文所有通用倾向（撞了以性子为准），但**铁律封顶**；交还"最先在意什么 / 推进快慢 / 情绪冷暖 / 话多话少"四个维度给性子。②【角色扮演链 → 心路 → 上手阶段】删每阶段写死的"标准问题清单"，改成"顺你这局性子自然冒那一两个在意点，别四种性子都奔同一个'什么皮、会不会塌'去"。③【随机池 → 性格卡】4 张卡按"风格旋钮"写差异锚点，尤其点明**每种人最先揪着的 1–2 个在意维度**（A 抠耐用值不值 / B 咬别家凭啥这价 / C 只淡看第一眼颜色感觉 / D 先盯好不好看配不配）。**提示词交还控制权 + 性格卡给锚点，缺一边都穿不透。**
- **来源**：coach v9 重构立项动因（6 局测试 4 性格同一行为指纹）。**警惕反作用**：交还控制权后冷淡型可能直接哑火 → 需在铁律给"话少也每轮推进一点、别缩成空回合"的下限。

### C7. 不识别"该收尾"的信号，原地空转复读

- **信号**：导购已把顾客在意的都答到位、顾客也没新顾虑了，但 Elena 不收尾、连续几轮原地打转（实测 loop10/11/12 连续 3 轮几乎一字不差"这皮质摸起来挺扎实"）；整场跑完也没出 `<?END_CHAT>`。
- **根因**：①提示词的心路⑤/收尾铁律写了"怎么收尾"，但**缺"什么时候该主动收尾"的判断信号**，模型靠感觉、容易错过；②"别重复刚说过的话"约束不够硬，没给"连续两轮原地打转=该推进或收尾"的明确触发。
- **落点与改法**：【角色扮演链 → ★铁律 → 第3条·一次一个意思/每轮推进】补"**连续两轮你都在原地打转、说同义的话，就是该往前推进或该收尾的信号**——要么抛一个真的新角度往下走，要么进入收尾两段式，别第三轮还在复读"。【心路 → 临门阶段】点明"导购把你在意的顾虑都答到位、你也提不出新的真顾虑了 = 你其实已经满意，该走收尾，别硬找话题空转"。配合【铁律 → 结束机制】的两段式 `<?END_CHAT>`。
- **来源**：2026-06-25 测试 loop10–12 连续复读 + 整场无收尾，用户列为问题⑤。

## 第二部分：场景档案

### 场景1·质感自用 Elena（角色扮演链）｜门店接待 · 销售主导 · 过程训练

- **类型**：门店导购接待、顾客（Elena）对 SA 话术做反应、练话术（高宽容、非知识考核）。系统**天然适配**（不像好买那种"客户主导"要反转主导权）。最像通用案例库里的"茶馆/仙草店内跟进（防御型客户）"+"太保锁单（角色冗长）"两个档案的组合。
- **Elena 人设硬约束**：27 岁武汉白领、真心爱包舍得为对的东西花钱、有主见识货不好糊弄、来线下亲手试背、**外部视角顾客（不查库不报参数，ADR 0004）**、只有觉得"被真听见"才聊得下去。
- **随机层**：最新后端模板口径统一为 `{rolePlayPersonality}` / `{rolePlayProduct}` / `{rolePlayState}`；场景1旧资料中可能仍见 `${roleplay_personality}` / `${roleplay_product}`，诊断时先按当前后端组装代码核对。性子是总开关、优先于正文通用倾向、但铁律封顶。
- **已确认的坑清单**：C1 状态连续性（头号、反复）、C2 旁白机制、C3 越界查库（已靠解绑 KB + 提示词治）、C4 看得见/可估的还问、C5 同担忧换皮、C6 性格层穿透、C7 收尾空转。
- **平台层（提示词改不了、反馈技术）**：话被截断（单次输出 token 上限/流式丢尾）、`<?END_CHAT>` 正则吃句尾、ASR 语音听岔（上语音后需识别置信度闸门）、角色扮演链解绑知识库/关 `search_knowledgebase`。

### 场景2·奥莱私域轻奢 Bella（角色扮演链）｜奥莱接待 · 销售主导 · 过程训练

- **类型**：Coach 奥莱/私域链路下的门店接待，顾客（Bella）预算更紧、第一次认真挑轻奢包，既想买到划算好包，又怕正品/价格/搭配踩坑。
- **Bella 人设硬约束**：26 岁武汉，刚毕业工作两年，月入六到八千；听朋友推荐来奥莱看 Coach，想要第一只拿得出手的轻奢包；不是产品专家，靠看、摸、问导购判断。
- **核心顾虑**：正品/渠道安心、划算价格、搭配闲置/买错风险。三者是底色，不是每局必须按顺序问完的清单；由本局行为卡决定先问什么、追多硬、何时放下。
- **状态账本**：Bella 的 repeated-question bug 高度依赖 `<?STATE 关切=...；动作=...；表态=...>` 是否稳定写出并被 `{rolePlayState}` 消费。正品/价格/搭配/退换/容量等都要按底层意思消耗，不能换皮重问。
- **随机层**：借鉴 `agency-agents` 的角色卡结构，Bella 性格池不扩成大量人物，而是 4 张高差异行为卡；每张卡固定 `说话气质 / 最先关切 / 证据阈值 / 追问策略 / 被打动信号 / 被推开信号` 六段，并必须带追问后的出口动作。推荐独立 config_key（见 ADR 0006），避免和 Elena 混抽。
- **已确认的坑清单**：正品/价格/搭配被模型当成无限清单、问过的问题换说法再问、提示词正文暴露变量/程序视角、`<?END_CHAT>` 和 `<?STATE ...>` 都叫"标签"导致收尾状态断档、C-怕踩坑型被轻微催单误触发提前结束、`rolePlayState` 里的 `关切=正品` 被误读成待办清单、"正价/专柜/做工/不对版"绕开正品消耗规则重新发问、产品卡色名被提示词示例带偏成近似营销色名、A-精算型把划算价格拆成当前价/正价差/到手价/底价多轮连续追问、材质名/绒面清洁/护理服务被拆成 `打理耐用` 多轮复问、性格卡差异被放大成 A 砍价/B 抬杠/C 全风险/D 补清单。

## 第三部分：coach v8.x→v9 迭代史沉淀（已验证的判断）

- **"伪失忆"根因不是模型笨、是提示词缺状态约束**：直调模型每轮喂全量历史，复读/重来是缺"认历史"硬约束。→ C1。
- **"一边禁一边种草"**：逐字列出要禁的台词＝把它反复喂进上下文、抬高被复制概率（v8.5 点名禁"随便看看"反而复发）。改法＝拆牌（删可照抄的台词模板）+ 类别/行为描述替代逐字列举。→ 调优铁律2。
- **复发只在"无效输入轮"**：导购那句打错字/没说清/答非所问时，模型接不住就退回兜底姿态。→ 给"没接住一句≠进度清零"的出口（铁律2）。触发源是打字错字（非语音），故未出 ASR 单；上语音后需平台识别置信度闸门。
- **角色扮演链去 KB**：真实 trace 显示挂 KB 让顾客复述卖点、被污染、34s/1.8 万 token。→ C3 + ADR 0004。
- **双模型对比（max vs plus）**：plus 那局"拉开看了内里还问有没有隔层"（max 没犯，靠模型自觉）= prompt 缺口，不能赌模型自觉、要写死。→ C4。
- **随机性格被压平**：5 阶段写死问题清单 + 逐字示范 + 固定画像引力把 4 性格拉成同一行为指纹。→ C6，靠"提示词交还控制权 + 性格卡给差异锚点"两层咬合治。
- **话被截断/省略号/旁白情绪泄露**：部分属平台层（token 上限）、部分纯 prompt 接近天花板，按既有结论挂评估链侧或反馈技术，别在角色扮演链硬抠。
