# Coach 陪练场景调优

把 coach 门店销售陪练里那个 AI 顾客（Elena）从"能跑"调到"像个真人"。核心动作：**发现问题 → 定位到提示词/随机池的哪一块 → 给出可直接落地的改法**，用业务侧看得懂的话、三件套格式输出。

> 本 skill 是通用 `ai-sparring-scene-tuning` 在 coach 客户的萃取定制版。两者最大区别在**架构**：通用版面向百炼 Agent 2.0（系统提示词固定不可动、主战场是改变量）；**coach 的角色扮演链已切回直调模型（ADR 0005）——后端组装"全量提示词"、整段都是我们自己写**，所以 coach **没有"不可动的系统层"那条前提**，提示词正文就是主战场。下文的层次模型按 coach 实际重写。

## 三条不可动摇的前提（coach 版）

1. **站在真实顾客角度审一切。** 判一句 Elena 说得对不对，唯一标准是"一个真心来挑包、有主见、识货的 27 岁白领，在这个情境下真会这么想/这么说吗"。出现"销售应该…""为了让 AI 好执行…"就是视角错了。
2. **先看是"约束缺失"还是"模型波动"，不要追影子。** coach 角色扮演链是**直调模型 + 每轮喂全量历史**，绝大多数"失忆/重复/出戏"不是模型真笨，而是**提示词缺一条显式约束**（直调没有 Agent 的隐式上下文兜底，全靠提示词把规矩写死）。但偶发一两次的也可能是模型波动——稳定复现（3 次见 2 次以上）才动手，单样本先复跑确认。
3. **输出永远是"改哪里、改成什么、为什么有用"三件套，且定位到具体落点。** 不甩整段提示词、不长篇大论。落点用业务侧说法（如【角色扮演链 → ★铁律 → 第5条·产品事实边界】、【随机池 → 性格卡 C-冷淡型 → content】），别只丢英文字段。映射见 `references/coach提示词结构与变量映射.md`。

## 必读：coach 架构认知（诊断前先装好这张图）

coach 陪练**规划上是三条链路**，但**当前只深耕角色扮演链**，评估两链已起草、未上线（用户明确"专注当前阶段、别急转下一阶段"——诊断默认聚焦角色扮演链，除非用户点名评估）。

- **三条链路**：①角色扮演（AI 演 Elena 跟 SA 对话，**当前主战场**）②实时评估（对 SA 最新一句实时评分，草案未上线）③整体评估（对话后综合评分+建议，草案未上线）。每条本质都是大模型读"替换变量后的提示词"→产出结果——这是"链路试跑"可行的根据。
- **coach 的两层 + 一个平台层**（取代通用版的"系统层 vs 变量层"）：
  - **L1 全量系统提示词**＝角色扮演链提示词正文（线上 SSOT＝`场景1_质感自用Elena/agent最新使用提示词.md`，**受保护**）。这是**主战场**，整段我们自己写。结构分块见映射文件。
  - **L2 注入变量**＝随机池抽取后注入正文的两个占位：`${roleplay_personality}`（今天的性子）、`${roleplay_product}`（一眼看上的那只包）。来自随机池 SSOT（`公共_跨场景复用/随机机制/最新随机池内容.md`，**受保护**）。"性格跑不出差异"这类问题要**同时**看 L1（提示词有没有把控制权交还给性子）和 L2（性格卡 content 有没有写出差异锚点）——两边咬合，缺一边都穿不透。
  - **L3 平台/后端**：直调百炼模型 API（qwen3.x-plus）、后端组装全量提示词并注入变量、`<?END_CHAT>` 收尾正则、单次输出 token 上限、ASR 语音听岔。提示词改不了的（话被截断、END_CHAT 吃句尾、token 上限）→ 标"建议反馈技术"，别在提示词里硬怼。
- **改 L1/L2 = 改线上百炼资产**，二者都在 `.Codex/protected-paths.txt`，**动手前先停下问用户**、不要自行绕过 guard。

## 四种工作模式 + 缺输入就索要

先判用户要哪种（可并用），核对必需输入，缺就先要——除非用户说"没有/不用了"。

### 模式一：测试诊断（从测试结果反推问题）——最常用
- **做什么**：从导出的对话数据（.xlsx）/截图/对话记录反推问题，走"现象→根因→落点"。
- **必需输入**：①测试记录（对话逐轮内容，理想含 loop 轮次、旁白、SA 输入与 Elena 回复）②本局抽到的性格卡/产品卡（用接口 `getRoleRandomData` 或 DB 取，知道这局是 A 理性还是 C 冷淡，才能判性格层有没有穿透）③当前线上角色扮演链提示词。
- **缺失处理**：只给现象没给提示词→索要当前提示词正文；只给一两轮→提示多跑几次确认稳定复现。
- **读法**：.xlsx 用 Python+openpyxl 读全（见 `references/coach场景档案与坑模式.md` 里的读取片段），按 loop_index 逐轮还原对话，**逐轮标注**"这轮 Elena 犯了哪条坑、对应坑模式 C几"。

### 模式二：提示词结构审查（对照规范审正文）
- **做什么**：把角色扮演链提示词正文逐块对照 coach 写作规范（`陪练场景搭建/0_提示词写作规范.md`）+ 本 skill 坑模式，查自洽性、查有没有把随机性格压平、查铁律有没有互相打架。
- **必需输入**：提示词正文（替换变量前的模板）。理想再加随机池性格卡/产品卡（查两层是否咬合）。
- **典型产出**：哪块写法会触发哪个坑（如"5 阶段写死标准问题清单→4 性格趋同""逐字示范句→被当台词复读"）。

### 模式三：链路试跑（用真实提示词复现/验证）
- **做什么**：拿"替换变量后的全量提示词"跑一遍，**复现**疑似问题或**验证**改动是否生效，省去每次回真实系统。
  - **本地业务接口试跑（coach 推荐）**：coach 有本地业务系统，可直接调真实接口跑出最真实的 AI 顾客输出，比自扮演可信。接口与流程见 `references/coach提示词结构与变量映射.md` 的"本地接口试跑"段。
  - **自扮演试跑（兜底）**：把替换后的提示词当角色指令、自己扮 Elena 对给定 SA 话术回应，观察是否犯坑。
- **重要免责**：自扮演用的模型与生产不同，结果是**方向性参考**；稳定结论仍要在真实系统/接口多跑几次（呼应"不追影子"）。

### 模式四：经验沉淀/复盘（让 skill 越用越准）
- 一轮调优后，把**已验证有效**或**已确认稳定复现**的发现，整理成结构化条目记进 coach 调优经验台账（模板 `assets/调优经验台账模板.md`）。也可单独触发（"复盘一下""沉淀到台账"）。
- **开局先读台账**（若有），套历史经验、别和旧结论打架、别重复踩"无效尝试"。
- **只沉淀验证过的**，不记未验证猜测；试过无效的记进"无效尝试"避免重走。

## 通用诊断步骤

1. **读全部材料再动手**——对话数据 + 这局的性格/产品卡 + 提示词正文，三样对齐才看得出"是提示词缺约束、还是性格卡没差异、还是模型波动"。只看一份发现不了。
2. **先判这是哪条链路、谁的问题**：不满针对"Elena 说的话/演的动作"→角色扮演链；针对"打分/建议"→评估链（当前未上线，多半先记着）。
3. **coach 场景定性（决定后面判断）**：coach = **门店接待 · 销售主导**（SA 接待、Elena 反应，系统天然适配，不像好买那种客户主导要反转主导权）+ **过程训练**（练话术、高宽容，不是知识考核的刚性评分）。Elena 是**外部视角顾客**（不查知识库、不报参数，ADR 0004）——这条最容易被违反。
4. **稳定性判断**：稳定复现（3 次 2 现）才改；偶发提醒"先观察"。
5. **现象→根因→落点**：每个问题描述现象（引到具体轮次）、分析根因、定位落点。**定位前先对照 `references/coach场景档案与坑模式.md`** 的 C1–C7 坑模式——这次 5 个问题基本都能直接命中。

## 输出格式（强制）

每个问题三件套，同事讨论式、简洁：

```
【改哪里】<业务侧定位：链路 → 区块 → 具体条，如：角色扮演链 → ★铁律 → 第5条·产品事实边界>
【改成什么】<可直接落地的新内容或明确改法>
【为什么有用】<一两句说清解决了什么、为什么>
```

- 多问题按 **P0/P1/P2** 分级（P0=出戏/暴露 AI/核心机制失效等必修；P1=明显影响真实感；P2=锦上添花）。
- 给了全套材料的系统性诊断可出 `.md` 报告：场景定性 → 逐坑模式问题（现象/根因/落点）→ 汇总表 → 经验沉淀。
- 确属平台/后端层（提示词改不了）→ 不在提示词里硬怼，标"建议反馈技术"并说明原因。

## 调优铁律（贯穿判断）

1. **角色"演得不像/话太多/动作堆一串"，先改写法风格，别加"每轮最多说 N 句"这类输出规则**——那是在错误的层加东西。冗长/表演欲的根因几乎都在**画像/性格卡的写法**：激烈词 + 大段剧本 + 写死示例台词 = 激发表演欲。修法是**用"语感描述"（这人说话什么劲儿）替代"心理描述/示例台词"**。但 coach 有个例外：**状态连续性（认历史不重来）这类约束，恰恰必须在提示词里显式写死**——因为直调模型没有 Agent 兜底（见坑模式 C1）。"风格问题改写法、记忆问题加约束"，别搞反。
2. **改写法时警惕"一边禁一边种草"**：逐字列出"不许说的话"＝把那句话反复喂进上下文、反而抬高被复制概率（coach v8.x 真踩过）。要禁某句，**改成类别描述/行为描述**，别逐字复述要禁的台词。
3. **以业务逻辑为锚，不以"让 AI 好执行"为锚**：别为稳定把 Elena 改得技术化、丢了真人味。改完让业务老师/培训方过目。
4. **改完用"真实顾客视角"验证，并查有没有引入新问题**：标准是真人/培训老师看着合不合理。尤其 coach 几条坑会**相互激化**——例如硬禁"复读"后，模型可能"为了说点新的"去换皮重抛同一担忧（C5）；硬交还性格控制权后，可能冷淡型直接哑火（C1 下限）。成对的坑要**一起验**（可用模式三试跑）。
5. **你的建议不都对**：提醒用户用业务判断力把关，尤其受保护产物动手前先问。
