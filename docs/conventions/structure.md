## 陪练场景搭建 分层骨架（L1–L5）  [B 桶 → 注入 + review；见 ADR 0002]
`陪练场景搭建/` 按下列分层组织，**新场景照抄 L3 标准模板即可扩展**：
- **L1 项目级**：`0_项目总览.md`（总进度 + 决策日志台账）、`0_提示词写作规范.md`（业务写作方法论全文）。
- **L2 公共资产** `公共_跨场景复用/`：`实时评估链/`（评估链**按链分目录**、多数场景共用；三链里的整体评估开工时增 `整体评估链/`，不笼统堆一个"评估链"目录）。**随机机制不放公共层**，随场景归入各自 `随机机制/`（见 ADR 0007）。
- **L3 场景层标准模板** `场景N_<画像角色>/`：
  ```
  场景N_<画像角色>/
  ├── 最新使用提示词.md   ← 角色扮演链 SSOT（唯一在用，保留此名）
  ├── 随机机制/
  │   └── 最新随机池内容.md ← 本场景随机池 SSOT（性格/产品/配置说明）
  ├── 草案/                    ← 在写、未定稿的设计稿
  ├── 记录/                    ← 调优 / 测试 / 调研 / 验证 过程记录
  ├── 归档/                    ← 废稿、被取代的旧版
  └── 画像素材/                ← 该场景画像 / 人设输入素材（可选）
  ```
  **状态用目录区分（草案 / 记录 / 归档），不用文件名后缀**；场景根和 `随机机制/` 只留当前在用 SSOT。
- **L4 知识库**：`知识库搭建/` 独立一层（百炼 RAG 用，部分跨场景复用，保持独立）。
- **L5 参考**：`参考资料/` 只读素材（POC 历史场景、机制分析、初始 prompt）。
  - 证据：`docs/adr/0002-陪练场景目录分层骨架.md`；`陪练场景搭建/` 实际结构。

## 物料流向  [B 桶 → 注入 + review]
- **上游原料只进 `辅助资料整理/`**（全二进制素材、无 .md）；**工程化 .md 产物只进下游** `知识库搭建/` 与 `陪练场景搭建/`。原料喂下游时**复制而非移动**（如 China Persona 复制进场景的 `画像素材/`）。 [B]
  - 证据：`辅助资料整理/`（`find -name '*.md'` 为空，全 .xlsx/.docx/.png/.jpg）。

## 知识库分库与归属  [B 桶 → 注入 + review]
- **物理分库 = KB1~KB6 六库，单容器 + 双标签承载**（不建六个百炼库）：46 篇全传同库，三链用 标签A=KB编号 / 标签B=内容类别 限定召回范围。 [B]
  - 证据：`知识库搭建/Coach陪练知识库架构设计_v2.md`（六库 v2 定稿）；`知识库搭建/百炼知识库部署配置与命中测试清单.md`（建一个库 + 双标签）。
- **KB 文档粒度判据**：「这块知识会不会被某个下游环节独立检索」→ 会则独立成篇。KB 文档是**被检索 / 被提炼的参考材料**，不是被整段复制的内容块。 [B]
  - 证据：`知识库搭建/Coach陪练知识库架构设计_v2.md`（文档粒度约定）。
- **库内归属边界**："服务是什么 / 规则边界"（静态事实）→ KB6 门店服务规范；"销售时如何用服务打动顾客（话术 / 动作）"→ KB3 流程；GenZ 沟通红线与方法**统一在 KB3 单处维护**（不两处漂移）。 [B]
  - 证据：`知识库搭建/Coach陪练知识库架构设计_v2.md`（v2 关键边界规则、D13/D14 GenZ 合并）。

## 归档约定  [B 桶 → 注入 + review]
- **被认定为误判 / 废稿 / 被取代时才归档**，移入场景的 `归档/` 子目录与线上版隔离，并配套用 git 回退基于废稿产生的衍生内容，避免废稿污染事实链。 [B]
  - 证据：`陪练场景搭建/场景1_质感自用Elena/归档/`（旧版废稿）；`陪练场景搭建/场景1_质感自用Elena/记录/调优记录.md`（2026-06-24：误判稿已归档、对抗测试章节 git 恢复）。
