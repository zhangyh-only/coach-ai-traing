# 内容/写作约定（本项目权威全文）

> 本仓是 Coach AI 销售陪练的**内容工程仓**，无源代码。下列规则约束 KB 文档与 prompt 的**写作、命名、版本、契约**。
> 每条标 A/B/C 桶（见 `README.md`）：A=机器可判定（由 `scripts/verify-local.sh` 文档卫生检查兜）；B=需理解的语义约束（靠注入落地）；C=参照背景。
> 证据路径一律为仓库根相对全路径，便于按图核对。

## 命名与元信息  [A 桶 → verify-local 校验]
- **KB 文档命名** `KB[编号]-[分类]-[简短标题].md`：`KB` 大写、三段用 `-` 连接、无序号前缀。例：`KB4-异议-异议主表.md`、`KB2-参考-颜色与术语词典.md`。 [A]
  - 证据：`知识库搭建/Coach知识库文档生成Prompt模板_v1_2.md`（命名规范）；`知识库搭建/具体KB内容/KB1..KB6/` 全 46 篇实际文件名。
- **KB 文档头部固定五字段元信息**：文档编号 / 所属知识库 / 版本 / 更新日期 / 数据来源。 [A（存在性）+ B（取值得当）]
  - 证据：`知识库搭建/Coach知识库文档生成Prompt模板_v1_2.md`（元信息固定五字段）。
- **场景目录命名** `场景N_<画像角色>`（如 `场景1_质感自用Elena`）：画像角色用客户原始画像名（非内部黑话）；渠道等细节进目录内文档、不锁进目录名。"复制新场景 = 换画像、照抄 L3 标准模板"（见 `structure.md` 分层骨架）。 [B]
  - 证据：`陪练场景搭建/0_项目总览.md` 决策日志；`docs/adr/0002-陪练场景目录分层骨架.md`。

## 内容写作铁律  [B 桶 → 注入 + review]
> 业务侧**完整方法论**（含 why / 示例 / 三条核心设计原则：旁白只给客观可见、事实层锁死 vs 表达层放开、画像带情绪演化）见 `陪练场景搭建/0_提示词写作规范.md`；本段是注入给 AI 的精炼版，单一权威是前者。
- **零编造**：缺失的价格 / SKU / 期限 / 尺寸一律标 `【待客户补充】` 或 `【以门店主数据为准】`，画像映射不确定标 `【待人工确认】`，**禁止凭印象推断填值**。 [B]
  - 证据：`知识库搭建/Coach培训资料内容分析提示词.md`（零推断硬规则）；`知识库搭建/Coach陪练知识库架构设计_v2.md` 大量【待客户补充】回填项。
- **专有术语原样保留、禁翻译归并**：END / ALL / AAA / 价格三明治 / BFA / FABS / Expressive Luxury / Tabby / Brooklyn 等按原文，**不得意译或合并相近缩写**（FABS≠BFA）。 [B]
  - 证据：`知识库搭建/Coach培训资料内容分析提示词.md`（专有术语原样保留硬规则）；`知识库搭建/百炼知识库部署配置与命中测试清单.md` T8/T9 专测 FABS≠BFA 不串。
- **数据来源字段取值规则**：企业培训资料→原文件名（多文件顿号分隔）；口头提供→`Coach提供·类型(年Qx)`；暂无源→`【待客户补充源文件】`。 [B]
  - 证据：`知识库搭建/Coach知识库文档生成Prompt模板_v1_2.md`（数据来源字段规则）。
- **异议写法**：一律写「关切主题 + 潜台词 / 心理倾向」口语化短句（10–25 字），**绝不写完整台词**（防 AI 机械复读、丢自然感）。 [B]
  - 证据：`知识库搭建/Coach陪练知识库架构设计_v2.md`（异议铁律）；`知识库搭建/具体KB内容/KB4/KB4-异议-异议主表.md`。
