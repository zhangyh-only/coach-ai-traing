# 设计文档 · 陪练场景搭建 目录分层骨架（L1–L5）

- **日期**：2026-06-25
- **类型**：结构性重构（改动·高 —— 动目录结构、碰受保护定稿产物、连带改线上契约路径）
- **状态**：设计定稿，待转执行计划（writing-plans）
- **配套 ADR**：`docs/adr/0001-台账移出受保护名单.md`、`docs/adr/0002-陪练场景目录分层骨架.md`（随本次新建）

## 1. 背景与目标

`陪练场景搭建/` 目前是"场景1 单场景打深"形成的扁平堆叠：评估链草案、随机池、各类记录、归档混在场景1 目录或散落顶层。随着要扩到场景2/3…，缺一套**可复制的标准结构**。

**目标**：把 `陪练场景搭建/` 重组成 **L1–L5 分层骨架**，并把场景子目录**标准化成一个模板**——以后每开一个新场景，照抄这套结构即可（这就是"可扩展体系"）。`知识库搭建/` 作为 L4 独立一层，本次完全不动。

**非目标（YAGNI）**：不预建空的 场景2/场景3 目录（要用时照模板新建）；不动知识库；不改任何 .md 的正文内容（只移动 / 改名 / 新建骨架文件 + 同步 rig 产物里的路径）。

## 2. 设计决策（已与用户确认）

| # | 决策 | 依据 |
|---|---|---|
| D1 | **随机池 = 全局单一，放 L2 公共层** `公共_跨场景复用/随机机制/` | 随机池对接后端单一 `nstr_global_config`、所有场景共用，不属于某个场景 |

## 4. 场景标准模板（L3 — 新场景照抄）

```
场景N_<画像角色>/
├── agent最新使用提示词.md    ← 角色扮演链 SSOT（唯一在用，保留此名）
├── 草案/                     ← 在写、未定稿的设计稿
├── 记录/                     ← 调优 / 测试 / 调研 / 验证 过程记录
├── 归档/                     ← 废稿、被取代的旧版
└── 画像素材/                 ← 该场景的画像 / 人设输入素材（可选）
```

**核心约定**：状态用**目录**区分（草案 / 记录 / 归档），不用文件名后缀。场景根目录只保留"唯一在用"的 SSOT。

## 9. 验收标准（可执行）

重构 + 连带更新完成后，必须全过：

1. `bash scripts/verify-local.sh` → **exit 0**，硬拦项全绿（KB 命名 / 定稿产物存在 / 变量名契约——路径已更新到新位置）。
2. **无悬空指针**：`AGENTS.md`、`docs/conventions/*` 引用的路径全部 `test -e` 通过。
3. **guard 仍生效**：对新位置的受保护文件（如 `公共_跨场景复用/随机机制/最新随机池内容.md`、`场景1_质感自用Elena/agent最新使用提示词.md`）实测 `guard.sh` 仍 `exit 2`。

5. **git 历史保留**：移动用 `git mv`，`git log --follow` 可追溯。
6. 两个 ADR 已建、索引已更新。
