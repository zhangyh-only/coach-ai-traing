# 0005 - 角色扮演链切回直调模型（prompt v9 重构）

## 背景与问题

角色扮演链此前发布为百炼 **Agent 2.0**，挂 Agent 的初衷是调知识库。两个变化使 Agent 失去存在必要：
1. **ADR 0004 已去掉 KB 检索**（AI 顾客不查库、事实核查归评估链）——Agent 的核心用途没了。
2. **Agent 模式长期"记不住上下文"**——6 局测试 + 业务反馈显示：问过的重复问、换种问法再问，严重伤陪练真实感。

同时这 6 局数据还暴露**提示词层**的硬问题：4 种随机性格跑出几乎同一份行为指纹——开场 100% "随便看看"、每局都问材质 / 划痕 / 会不会塌 / 肩带、单条 150–220 字爱解释、把示范句原样复述、C 冷淡性格却最话痨。根因＝**提示词过度圈定**（5 阶段写死标准问题清单 + 逐字对比示范变台词 + 固定画像引力）把随机性格压平了。

## 决策

1. **调用方式**：角色扮演链**弃用百炼 Agent 2.0、切回「业务后端组装全量提示词 + 直调百炼模型 API（qwen3.7-plus）」**（不再经 Agent 应用编排）。每轮喂 `[系统提示词 + 全量历史]`、上下文完整无 RAG 打断；随机性格 / 产品机制与变量契约（`roleplay_personality` / `roleplay_product`、`{name,content,enabled}`）不变，由后端拼 prompt 时注入。
2. **prompt 重构为 v9**（四项）：① 结构 12 扁平一级标题 → **4 区 + 铁律置顶**；② **减负**（删对比示范 + 5 阶段标准问题清单 + 清掉嵌在铁律 / 心路里的台词种子）；③ **治话多**（铁律上限"一次一个意思 + 别超三句"＋下限"话少也每轮推进一点、防哑火"）；④ **随机风格层**（区二性子提权、**辖域钉死全文 + 铁律封顶**，把"**在意什么 / 推进快慢 / 情绪冷暖**"三维度交还性子，开场按性子分化）。
3. **随机池配套**：4 张性格卡按 **6 旋钮**（话密 / 姿态 / 挑剔 / 推进 / 情绪藏露 / **最先在意**）重写，与提示词区二、区四③咬合（提示词交还控制权、卡给差异化锚点，缺一边都穿不透）。

## 为什么

- **治"记不住上下文"**：直调把完整 `[系统 + 全历史]` 一次喂入，没有 Agent 框架的 RAG 召回 / 多步规划打断上下文；配合 v9 铁律"认历史置顶"（直调无 Agent 记忆兜底、全靠提示词 + 全历史）。
- **去 Agent 无损**：Agent 唯一刚需（调 KB）已被 ADR 0004 移除，切直调不丢能力、反而省 RAG 开销与延迟。
- **风格层落地需要直调的可控性**：v9 把性子提权、辖域钉死，需要完整提示词每轮稳定喂入，直调比 Agent 黑盒更可控。

## 备选方案

- **保留 Agent、仅改提示词**：不选——"记不住上下文"部分源于 Agent 框架对历史的处理，且 Agent 已无 KB 刚需，留着徒增黑盒与开销。
- **直调但不重构提示词**：不选——切调用方式不解决"4 性格趋同"，那是提示词层问题，必须同批 v9 重构。
- **第一版 v9 只声明"性子优先"**：不选（经 6 视角红队否决）——一句声明压不过密集的具体引力（统一画像 / 最高优先级的铁律 / 整页热络示范），差异穿不透；必须把三维度从结构上交还性子 + 性格卡给锚点。

## 影响与代价

- 正面：上下文完整（治重复问）、风格层可落地、省 RAG 开销 / 延迟、三链职责清晰（角色扮演演真人、评估核事实）。
- 代价：① **后端 / 平台侧**需把角色扮演调用从百炼 Agent 应用改为**代码直调百炼模型 API（qwen3.7-plus）**、解绑 KB、后端组装全量提示词并注入变量、重灌 `nstr_global_config`（后端动作，AI 改不了）；② 随机池受保护改动（开关代改、已授权、改完恢复 guard）；③ 机制说明"走 agent"描述待后端直调落地后同步（已加注记）。
- 风险：① 直调全靠提示词认历史、无 Agent 记忆兜底——靠 v9 铁律1强化 + 实测验证；② 风格层"声明→落地"靠提示词 + 性格卡两边咬合，缺一边穿不透——已同批改、对抗自测两侧覆盖；③ v9 先落草案、未替换线上 v8.4——替换按受保护流程单独走。
