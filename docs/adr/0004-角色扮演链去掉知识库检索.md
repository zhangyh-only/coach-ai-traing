# 0004 - 角色扮演链去掉知识库检索

## 背景与问题

自 v8.3 起，角色扮演链提示词《涉及产品事实，先查资料再开口》要求 AI 顾客"只要导购说到产品事实，先查随附知识库，判断他说得对不对、有没有夸大"。百炼角色扮演 Agent 据此挂了知识库、AI 顾客每轮触发 `search_knowledgebase`。当时把"KB 真用、查证戳穿撒谎"当作亮点。（注：同日稍早记的 ADR 0003 仍沿用"产品事实被问才查 KB"的旧假设——彼时尚未观测到本 ADR 记录的副作用；本 ADR 据真实 trace 翻转该子假设。）

2026-06-25 用户拿一条**真实 trace**（导购"看看这款 Tabby，卖得好"→ 顾客"独一无二人家都这么说嘛……容量的话手机短钱包粉饼口红钥匙……这中号能装得下不？"）核对链路日志，暴露三个硬问题：

1. **角色违和（根本）**：顾客本是**外部视角**——只知道看到的、摸到的、导购说的。RAG 给了它**上帝视角**：能看到产品内部资料、卖点话术，连"【待客户补充】"这种内部标注都检索得到。真人顾客不会去翻产品库。
2. **内容污染、来源不可见**：顾客把 KB 卖点话术"独一无二"当自己的话复述（"独一无二人家都这么说嘛"——顾客凭什么知道品牌怎么夸产品）；把 KB 里"容量参考：手机/墨镜盒/笔记本/签字笔"改写成自己的物品清单。这些来源对 SA（被陪练的销售）完全不可见，破坏真实感。
3. **开销爆炸**：KB 检索返回 **75 个节点**全量注入上下文，单轮 **34.66 秒 / 1.8 万 token**（最终生成那次 input 就 1.1 万 token）。为"演顾客说一句话"付出 RAG + 重生成的双倍开销。

## 决策

**角色扮演链去掉 AI 顾客的知识库检索**，并把产品事实核查职责明确归到评估链：

1. **顾客回归"外部视角"**：产品事实（精确尺寸容量、皮料学名/产地、工艺、认证、保养等）**事先不知道、不查库、不报参数**，靠看 / 摸 / 问导购；识货靠常识追问（"你光说好到底好在哪""能装下我那些零碎不"），**不负责当场核对导购参数报得准不准**——就是个普通顾客。
2. **事实核查归评估链**：实时评估 / 整体评估本就 RAG 读 KB，判断 SA 说得对不对、有没有夸大。三链关注点分离——**角色扮演 = 演真人，评估 = 核事实**。
3. **落地两处**：① 提示词草案《涉及产品事实，先查资料再开口》**整节重写**为《涉及产品事实，你只知道看得见摸得着的》（草案第三批 G，并吸收原 D 术语边界）；② **百炼侧：角色扮演 Agent 解绑知识库 / 关 `search_knowledgebase` 工具**（平台配置，用户控制台操作）。

## 为什么

- **真人感**：顾客不读产品库，才像顾客。"独一无二""容量清单"这类只可能来自资料的话不再从顾客嘴里漏出来。
- **关注点分离**：事实核查是评估链的本职（它读 KB、它打分），不该让"顾客"越俎代庖。让每条链只干自己的事。
- **性能**：去掉每轮 RAG + 重生成，单轮延迟 / token 大幅回落。
- **"识货"不丢**：顾客的识货感靠人设（有主见、识货、不好糊弄）+ 常识追问演出，不需要精确查库——真人也是凭经验质疑、不是现场翻资料。

## 备选方案

- **保留 KB、大幅收敛**（限触发条件 / 限返回节点数 / 只取核心字段）：不选 —— 治标不治本，"顾客查库"的角色违和与"漏卖点话术"的污染仍在，只是程度轻些。
- **开局一次性预置精简产品认知给顾客**（不实时查、开局塞一段顾客"该有的认知"）：不选 —— 与 ADR 0003"产品事实守 KB 单一源、不写进随机池"冲突，且"顾客该预先知道多少"难界定、易又编出错。

## 影响与代价

- 正面：真人感↑、单轮延迟 / token↓、三链职责清晰。
- 代价：① 提示词草案改（已落第三批 G）；② **百炼侧需解绑知识库**（用户控制台操作，AI 改不了）；③ 修订 ADR 0003"被问才查 KB"那句（已加修订注）。
- 风险：顾客不再"当场戳穿导购的具体参数错误"（v8.3 曾视为亮点）—— 但这本是评估链职责、评估链已 RAG 覆盖，陪练价值不丢；需在**评估链验收时确认事实核查确实接住**（SA 夸大 / 说错能被评估抓到）。
- 取代关系：取代 ADR 0003 决策里"产品事实被问才查 KB"的**子假设**（查 KB 的从"顾客"改为"评估链"）；ADR 0003 其余（身份可指认 / 事实不写池 / KB 单一源）仍成立。

## 后续

- 草案第三批 G 待百炼回归测，验收点：① 观测里**不再出现 `search_knowledgebase` span**；② 顾客不复述"独一无二"等卖点话术；③ 单轮延迟 / token 显著回落。
- 百炼侧：角色扮演 Agent **解绑知识库 / 关 `search_knowledgebase` 工具**。
- 评估链验收时确认"事实核查"覆盖到位（导购夸大 / 报错参数能被评估识别）。
