# 0003 - 产品身份可由旁白指认 / 产品事实守 KB 单一源

## 背景与问题

2026-06-24 随机池重构（见 ADR 0002 子决策、随机池设计原则）定下：产品池 `content` 只写外观口语 + 末尾〔本局对应款〕锚点，**锚点"绝不对学员报型号"**；提示词侧《看不见的动作》段也写"提到具体某只包……**别报型号尺寸**"。本意是保"真人感"（真人挑包不会张口"我要看 Tabby 26"）。

2026-06-25 业务老师测 11 条反馈 + 用户复盘暴露副作用：**SA（被陪练的销售学员）不知道顾客在谈哪只具体的包**——顾客既不报款式、旁白也不点明，导购无从判断该介绍哪款、用哪套话术，流程推不动、不自然。用户原话："你现在不说具体产品信息，SA 还是不知道要谈论具体哪个产品。"

## 决策

把"绝不报型号"**反转**为：**产品"身份"可由旁白指认，产品"事实"仍守知识库单一源**。落到四处：

1. **提示词《看不见的动作》整节重构为《旁白机制》**：把括号定位成"演给导购看的呈现层"，**输出形态（括号）与克制原则不变**；新增正当用途——旁白里可点明在看哪只具体的包（款式 / 型号）。
2. **放开"别报型号"**：旁白可点款式 / 型号让 SA 看清；顾客**对话口语**仍用"那只焦糖棕的""那个黑色方包"自然指代（保真人感）。
3. **随机池〔本局对应款〕语义升级**：从"绝不说出口的内部锚点"→"可由旁白点明的款式身份"；6 张产品卡名去掉"勿报型号"标注（`content`/`enabled`/契约字段不动）。
4. **机制说明同步**：产品卡 `content` 说明改为"款式身份可由旁白点明、对话口语指代；产品事实走知识库"。

**边界（关键）**：放开的只是"**哪款包**"的指认；**产品事实细节（皮质、精确尺寸、价位、工艺）仍不写进随机池**。此边界与同批草案改动 D（顾客不主动飙"头层牛皮"等材质术语）**并行不冲突**——D 管"事实/参数怎么说"，本决策管"身份/哪款能不能指认"。

> ⚠️ **2026-06-25 修订（见 [ADR 0004](0004-角色扮演链去掉知识库检索.md)）**：本条原写"产品事实……被问才查 KB"，假设由 AI 顾客自查知识库。真实 trace 实测发现"顾客自查 KB"会复述卖点话术、被资料污染、单轮开销爆炸——故 ADR 0004 决定**角色扮演链顾客不再查 KB**，产品事实核查改归评估链。"产品事实守 KB 单一源"仍成立，只是查 KB 的角色从"顾客"变成"评估链"。

## 为什么

- **陪练可用性 > 过度真人感**：训练价值的前提是 SA 知道谈论对象。"绝不报型号"把"真人感"保过了头，牺牲了流程可推进性。
- **拆开"身份"与"事实"两件事**：身份（哪款）放开、由旁白指认即可锚定，不伤真人感（旁白本就是呈现层、非顾客台词）；事实（参数）继续守 KB 单一源，既防顾客飙参数失真、又防产品事实在随机池与 KB 两处漂移。

## 备选方案

- **维持"绝不报型号"**：不选 —— SA 锚定不了谈论对象、流程卡，正是本次要解决的痛点。
- **让顾客对话口语直接报型号**（"我要看 Tabby 26"）：不选 —— 真人挑包不这么说话、伤真人感；改用"旁白点明 + 口语指代"兼顾锚定与自然。
- **把产品事实也写进随机池让顾客直接讲**：不选 —— 破坏 KB 单一源、产品事实两处漂移，且顾客飙参数不真实。

## 影响与代价

- 正面：SA 能锚定在谈哪款、流程更自然可推进；产品事实仍单一源（KB），不漂移。
- 代价：改了受保护 SSOT（随机池 + 机制说明）——本次经用户授权"开关代改"临时放行、改完即恢复 guard、`verify-local` + JSON 解析校验全绿。提示词改动在 [状态连续性约束草案](../../陪练场景搭建/场景1_质感自用Elena/草案/agent提示词_状态连续性约束_草案.md)（拟 v8.5），未动线上 SSOT。
- 风险：旁白点款若**过频**，可能略损真人感、或与"旁白要克制"原则打架 —— 靠《旁白机制》"大多数回合不加括号、一轮最多一句"约束，并在百炼回归测时**观察旁白点款频率**。

## 后续

- 草案 v8.5 待百炼回归测；测时专项观察：① SA 能否稳定知道在谈哪款；② 旁白点款是否过频/出戏；③ 顾客是否仍不飙材质参数（改动 D 不被本决策带偏）。
- 定稿替换线上 SSOT（`最新使用提示词.md`）时，按受保护流程走（先停下问用户 + code-reviewer 复核）。
- 产品事实（如 Tabby/Brooklyn 各款精确容量、尺寸）仍以 KB / 门店主数据为准，随机池不回填。
