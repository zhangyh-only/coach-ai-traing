# 计划：搭建场景1（Elena 正价）整体评估链系统提示词

> 本计划由 superpowers brainstorm → writing-plans 产出，套本仓 `docs/plans/_template.md`。
> 本仓为纯 prompt 工程仓：「实现」＝写 prompt，「测试」＝`scripts/verify-local.sh` 文档卫生检查 + 对抗式自测（造完整对话喂 prompt），真验收在百炼侧人工跑。
> 执行前已与用户多轮对齐方案并放行（见下「已定决策」）。

## 目标
产出场景1（质感自用 Elena·正价）可在阿里云百炼搭建运行的**整体评估链系统提示词**：SA 与 AI 顾客**完整对练结束**后，读**全程对话**，按 Coach 四维度（专业性 30 / 同理心 25 / 沟通技巧 25 / 流程意识 20）**打分**（分只到维度级、**报告不出总分**），三档定级（优秀/良好/待提升）、证据导向，知识层走 RAG（必引 KB5、按需 KB2/KB3/KB4），ASR 容错，输出**严格照 POC 示例模板**（四维度块 + 关键问题汇总 + 综合反馈）。形态＝**通用骨架 + 场景1 适配块**，与实时评估链**同一把尺、同构**，本链＝其「打分 + 全程复盘」版。产物为 **L2 公共资产**。

## 已定决策（brainstorm + 用户微调拍板，作为本计划前提）
1. **报告主轴＝四能力维度**（非"五销售环节"——上一轮口误已修正）；以用户提供的 Coach POC「产品介绍」整体评估 prompt 为**格式蓝本**。
2. **维度命名/权重/档位**取实时评估链 / KB5 口径（专业性 30 / 同理心 25 / 沟通技巧 25 / 流程意识 20；三档区间取 **KB5-评分**：专业 25-30/20-24/0-19，同理·沟通 21-25/16-20/0-15，流程 17-20/13-16/0-12）；**维度名用短名**（用户：名字别太长）。
3. **分值只到维度级**，维度下小考点（评估要点）只在「表现分析」里说、不单独给分；**报告不输出总分**（用户要求；恰与 POC 示例模板天然一致——示例本就无总分行）。
4. **评分基调＝核心严、细节宽**（用户折中选项）：扣分情形 / 红线**严格执行**；产品简称、ASR 错字、口语等价、表达风格差异**宽容不扣**；"培训鼓励"原则不写进报告。
5. **知识层 RAG 化**（必引 KB5、按需 KB2/KB3/KB4，**不内联**事实表，区别于 POC 全量塞 prompt）；**优化建议里的话术示例必须来自检索到的 KB2/KB3**，检索不到只给方向、不编（用户：话术一定要符合实际 KB）。
6. **扣分注意兼容性**（用户要求）：① 只评对话实际发生的阶段、没走到的不倒扣；② 区分"有机会没做"vs"没给机会"；③ 同一行为不重复扣；④ **不因结果（成交与否）扣分**；⑤ 性子差异（理性/比价/冷淡/心动）兼容。
7. **输出模板严格照 POC 示例**（`### 销售维度评估` → 四个 `#### 维度（X/X分）`+`<?END_DIMENSION>` → `### 关键问题汇总` → `### 综合反馈`）；识别 `<?END_CHAT>` 为"读全程"的结束边界（不评标签本身）。
8. **产物落 L2** `公共_跨场景复用/整体评估链/`（新建该目录，与 `实时评估链/` 平行；依 ADR 0002 评估链按链分目录、跨场景共用）。

## prompt 模块结构（一份文件内的章节）
| # | 模块 | 职责 | 主要来源 |
|---|------|------|---------|
| 1 | 角色与核心铁律 | 整体评估定位；读全程评整体 / 四维度打分分到维度级·不出总分 / 证据导向 / 事实以 KB 为准 | POC §角色 + 实时评估链铁律 |
| 2 | 输入格式说明 | 完整对话数组；评估对象=SA 全程；`<?END_CHAT>` 作结束边界；旁白=情境；短对话/提前结束只评已发生 | 实时评估链 + 角色扮演链结束机制 |
| 3 | 评估依据 | ①ASR 容错 ②术语等价/产品简称识别 ③RAG（必引 KB5、按需 KB2/KB3/KB4；话术来自 KB、不编） | POC 容错段 + 实时评估链 RAG 段 |
| 4 | 四维度尺 | 四维度好/差速记（引 KB5）+ 评估要点(小考点) + 三档区间 + **扣分情形** + **兼容性原则** | KB5-维度 + KB5-评分 + POC 扣分情形 + 用户微调6 |
| 5 | 红线与禁止指标 | 五类绝不生成的评估项；Coach 红线落对应维度扣 | KB5-设计 + 实时评估链 |
| 6 | 评分基调 | 核心严、细节宽；培训鼓励不写进报告 | 用户微调4 + POC 宽容原则 |
| 7 | 整体评估五步 | 通读全程→分维度收证据→定档给分→提炼关键问题→综合反馈 | 本链定位 |
| 8 | 生成规范 | 表现分析（分点/优点为主/禁刻意转折/引用≤15字）/ 优化建议（来自KB/≤50字/不出折扣/不重复建议）/ 关键问题（1-3个/不新增）/ 综合反馈（3-5句） | POC 生成规范 + 用户微调 |
| 9 | 输出模板 | 严格照 POC 示例 + 维度短名 + 无总分 + `<?END_DIMENSION>` | 用户微调5 |
| 10 | 本场景固定信息块 | Elena 画像 / 正价渠道 / 五阶段流程实例 / 事实锚点 / 旁白处理（复制新场景只换这块） | 角色扮演链 + 实时评估链固定块 |

## 步骤
1. **新建目录 + 写系统提示词模块 1–3**：创建 `公共_跨场景复用/整体评估链/整体评估链_系统提示词_草案.md`，写角色与铁律、输入格式、评估依据（ASR + 术语等价 + RAG）。
2. **写模块 4–6**：四维度（判据 + 评估要点 + 三档 + 扣分情形 + 兼容性原则）、红线与禁止指标、评分基调。
3. **写模块 7–10**：整体评估五步、生成规范、输出模板（照 POC 示例）、本场景固定信息块（Elena）。
4. **通读整合 + 卫生检查**：与实时评估链口径对齐（四维度名、阶段名、Tabby/颗粒纹/防泼水锚点）、确认**未误用** `roleplay_*` 变量、确认输出模板**无总分**；跑 `bash scripts/verify-local.sh` 硬拦项不退化。
5. **写对抗自测清单**：`整体评估链_对抗自测清单.md`，14 条完整对话用例（含兼容性 / 不因结果扣分 / 无总分 / 话术来自 KB）。
6. **回写台账**：`0_项目总览.md` §6/§9/§10/§13。
