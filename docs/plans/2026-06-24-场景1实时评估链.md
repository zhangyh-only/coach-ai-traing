# 计划：搭建场景1（Elena 正价）实时评估链系统提示词

> 本计划由 superpowers brainstorm → writing-plans 产出，套本仓 `docs/plans/_template.md`。
> 本仓为纯 prompt 工程仓：「实现」＝写 prompt，「测试」＝`scripts/verify-local.sh` 文档卫生检查 + 对抗式自测（造 SA 发言喂 prompt），真验收在百炼侧人工跑。
> 执行前已与用户对齐三项骨架决策（见下「已定决策」），用户已批准方案。

## 目标
产出场景1（轻奢包追求者 Elena·正价）可在阿里云百炼搭建运行的**实时评估链系统提示词**：SA 对练中每一轮、只评最新一条销售发言、**不打分**，用 Coach 四维度统一尺 + 阶段动态侧重判断，知识层走 RAG（必引 KB5、按需 KB2/KB3/KB4），事实轻核对 + 标疑点，ASR 容错，输出对准 R3 给"可直接上柜台"的参考话术。形态＝**通用骨架 + 场景1 适配**，与角色扮演链 v8.x 配套。

## 已定决策（brainstorm 拍板，作为本计划前提）
1. **落地方式**＝骨架移植 + RAG 化：以用户提供的「标准产品实时评估 prompt」为骨架蓝本（比历史 POC `历史场景.md` 那份更精炼、关联资料天然外挂＝对应 RAG），知识层从内联改为 KB 检索。
2. **事实核对**＝轻核对 + 标疑点：硬伤当场点出给方向；KB 查不到不强求、转评技巧；"KB 没有 ≠ 学员编造"不倒扣；价格/SKU 标"以门店主数据为准"。
3. **评判尺子**＝Coach 四维度（专业性/同理心/沟通技巧/流程意识，引 KB5-维度）统一标尺 + 按本轮销售阶段动态侧重主考维度。
4. **输出带一句很轻的「当前阶段」标注**（如 `〔触感体验·重专业性〕`）。
5. **场景1 画像写死**（Elena＝质感自用），不留 `${}` 变量口子（实时评估是按场景部署的独立 agent；与角色扮演链每局随机的 `${roleplay_personality}` 不同）。

### 对抗式自测 14 条用例（验收锚点，落入清单文件）
> 每条＝构造一段对话数组（含 1+ 轮历史 + 最新一条 SA 发言）→ 期望评估行为。

1. **专业性·事实正确**：触感阶段，SA 正确讲焦糖 Tabby 颗粒纹来源/工艺 → 只出「做得好」，不挑刺、不硬凑建议。
2. **专业性·事实硬伤**：SA 说"Tabby 26 能装 iPad" → 当场点出与 KB 冲突 + 给正确方向，落专业性。
3. **ASR 容错**：SA 说"这个泰比包是颗粒文皮" → 不因"泰比/颗粒文"扣分，按 Tabby/颗粒纹理解，输出不出现纠错说明。
4. **不跨节点**：异议处理轮，SA 只答异议未做产品完整介绍 → 不要求补"产品介绍"要素。
5. **红线·夸大**：SA 说"这包完全防水、绝对不会刮" → 点出夸大红线（防泼水≠完全防水），落专业性。
6. **红线·GenZ 禁忌**：破冰轮 SA 贴身硬推 / 未问先报折扣 → 落沟通技巧，提示给空间 / 不主动提折扣。
7. **同理心·阶段侧重**：需求诊断轮 SA 不提问直接推款 → 主考同理心，提示开放式提问升到使用场景。
8. **流程意识·成交时机**：顾客已发购买信号，SA 没接、继续平铺介绍 → 主考流程意识，提示识别信号、适时促成。
9. **R3 话术打动力**：SA 回答正确但干巴 → 参考话术给贴 Elena/GenZ 口吻、可直接照搬的话术（非书面参数）。
10. **好·不挑刺**：SA 本轮表现完美 → 只出「做得好」，不硬凑「优化建议/参考话术」。
11. **差·只问题**：SA 答非所问/敷衍只回"嗯" → 不出「做得好」，只给「优化建议+参考话术」。
12. **不打分**：任意轮 → 输出**不含**任何分数/总分/档位数字。
13. **旁白处理**：顾客台词带旁白"（她拿起包看了看）" → 旁白当情境不当 SA 的话评；但"SA 有没有接住顾客信号"算同理心/流程观察点。
14. **价格主数据**：SA 报具体价格 → 不臆造价格对错，参考话术/建议里标"以门店主数据为准"，不出折扣。
