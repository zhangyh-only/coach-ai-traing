# 场景1 角色扮演链 v9 重构 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: 用 superpowers:subagent-driven-development（推荐）或 superpowers:executing-plans 按 task 逐个实施。步骤用 checkbox（`- [ ]`）跟踪。
> **配套 spec**：[设计文档](2026-06-25-场景1角色扮演链提示词v9重构.md)（背景诊断 / 决策 / ST1–ST4 / 验收）。本计划只管"一步步怎么做"。

**Goal：** 把场景1 角色扮演链从百炼 Agent 切回「业务后端组装全量提示词 + 直调百炼模型 API（qwen3.7-plus）」，并把提示词重构为 v9（结构分层 + 减负 + 随机风格层），随机池性格卡同步按"风格旋钮"重写，使不同性格跑出可感差异、单条更短、不再 100% "随便看看"。

**Architecture：** 提示词侧（v9 草案）+ 随机池侧（性格/产品卡）两套资产**配套同批改、同批验**；提示词给随机变量提权+减负，随机池把 content 写成能穿透的风格差异。v9 先落草案，不动线上 SSOT；随机池为受保护产物，走授权流程。

**Tech Stack：** Markdown prompt（Obsidian vault）；运行底座＝**业务后端组装全量提示词、直调阿里云百炼模型 API（qwen3.7-plus）**（不经 Agent 应用），每轮喂 `[系统提示词 + 全量历史]`，`${roleplay_personality}`/`${roleplay_product}` 由后端注入；验证＝`scripts/verify-local.sh` 文档卫生 + `grep` 断言 + `python json.load` + 对抗自测清单 + 百炼模型人工真跑。

## Global Constraints

> 每个 task 都隐含遵守以下全局约束（值逐字取自 spec）。

- **全程简体中文**叙述；prompt 文案为中文口语。
- **继承 v8.5 草案全部能力**：状态连续性约束 / 旁白机制 / 放开报型号 / 去 KB 检索 / "随便看看"拆牌 / 输入容错。源文件 `草案/agent提示词_状态连续性约束_草案.md`。
- **守 ADR 0003**（产品事实不进随机池）、**ADR 0004**（角色扮演不查 KB / 事实核查归评估链）。
- **百炼变量契约不变**：`${roleplay_personality}` / `${roleplay_product}`；性格池 **4 张** / 产品池 **6 张**；每卡 `{name, content, enabled}` 字段齐、`enabled=true`。
- **不碰线上 SSOT** `最新使用提示词.md`（v9 只落草案；替换线上是后续单独动作）。
- **随机池 = 受保护产物**：`最新随机池内容.md`、`角色扮演随机机制说明.md`。改前临时放行 guard（注释 `.claude/protected-paths.txt` 对应行）→ 改 → **恢复 guard** → JSON 校验 → 收尾 code-reviewer。
- **verify-local 硬拦项必过**才算该产物 task 完成（`bash scripts/verify-local.sh` exit 0）。
- **不自动 commit/push**；提交待用户明确许可（用户全局约束优先于 skill 的 frequent-commit）。
- **本仓"测试"适配**：无 pytest/编译。创作类 task 的最终文案在执行时创作（可用 Workflow 多方案起草 + 红队对抗），其"正确性"由本计划的 `grep`/JSON 断言 + 对抗自测清单 + code-reviewer 把关，**非预先写死文本**——故各 task 给"改写规格 + 验证断言"，不是 placeholder。

7. 别输出排版符号 / 少用省略号。

### Task 8：随机池性格卡重写（受保护 · 授权）

**Files：** Modify `陪练场景搭建/场景1_质感自用Elena/随机机制/最新随机池内容.md`（性格池）；临时改 `.claude/protected-paths.txt`
**消费：** T3 风格旋钮契约。

**改写规格（4 张卡，按 6 个风格旋钮，去台词去分叉清单）：**
旋钮：①话密 ②姿态(热络/防备) ③挑剔度 ④推进节奏 ⑤情绪藏露 ⑥**最先在意(关注点)**——红队新增、治"问材质趋同"的关键：每张卡点明这种人最先揪着的 1-2 个在意维度，与提示词区四③"不同性子在意的天然不同"咬合（提示词交还控制权、卡给差异化锚点，缺一边都穿不透）。
- **A 理性**：话适中有条理 / 平和带审视 / 挑剔（导购含糊就明显不满意）/ 谨慎慢决 / 情绪藏着怕被拿捏 / **最先在意：耐不耐用、值不值、细节经不经得起推敲**。
- **B 比价**：话适中 / 较真不轻易服气 / 爱比较权衡（**风格化态度，不写"必问凭啥比别家贵"这种固定问题**）/ 要被说服才推进 / 情绪半露 / **最先在意：跟别家比、这只凭啥这价、设计做工差在哪**。
- **C 冷淡**：话少 / 防备往后躲 / 随和但慢热 / 慢、需要空间 / 情绪淡不外露 / **最先在意：第一眼的颜色 / 感觉对不对，不主动深挖、被问才淡淡说**。**删"先撂一句随便看看"**（具体台词，与开场分化冲突）。
- **D 心动**：话偏多 / 热络主动 / 随和好说话 / 看对眼就快 / 藏不住喜欢 / **最先在意：好不好看、配不配自己、什么场合背**。
- 每卡只写"这一局的性子基调"，不重复画像、不写流程指令（守随机池设计原则）；保留 `{name, content, enabled:true}`。

- [ ] **Step 1**：注释 `.claude/protected-paths.txt` 中随机池对应行（临时放行 guard）。
- [ ] **Step 2**：按规格重写 4 张性格卡 content。
- [ ] **Step 3**：验证 `python3 -c "import json,re; ..."` 性格池数组可解析、4 张、字段齐；`! grep -q "随便看看"` 在性格池段。
