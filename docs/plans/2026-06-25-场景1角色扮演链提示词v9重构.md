# 计划：场景1 角色扮演链提示词 v9 重构（切直调模型 + 结构分层 + 减负 + 随机风格层）

> 本计划由 superpowers brainstorm 产出，套本仓 `docs/plans/_template.md`。
> 本仓为纯 prompt 工程仓：「实现」＝写 prompt，「测试」＝`scripts/verify-local.sh` 文档卫生检查 + 对抗式自测（造完整对话喂 prompt），真验收在百炼侧人工跑。
> **主线＝调用方式切换（百炼 Agent → 全量提示词 + 直调模型）**，下挂四个并列子任务 ST1–ST4；不是单改随机变量。

## 背景与数据诊断

**由头**：角色扮演链此前发布为百炼 **Agent 2.0**，挂 Agent 的初衷是调知识库；ADR 0004 已决定去掉 KB 检索，Agent 不再有存在必要。更要紧的是——Agent 模式长期"记不住上下文"（问过的重复问、换种问法再问），故切回 **全量提示词 + 直调模型**：每轮 `[系统提示词 + 全量历史]` 一次性喂模型，上下文完整，配合 v8.5 状态连续性约束治"伪失忆"。

**数据诊断**（2026-06-25，6 局对练 / 覆盖 4 种性格，`对话数据.xlsx` + `对话记录对应变量数据.xlsx`）：
- **随机变量被压制**：4 种性格跑出几乎同一份"行为指纹"——开场"随便看看"**6/6 全中**（但 A/B/D 性格卡里根本没这句，它来自提示词阶段①）；每局都问"什么皮 / 划痕娇气 / 会不会塌 / 肩带长度"（来自《你会怎么一步步走》阶段③写死的标准关心清单）。
- **示范变台词**（最硬病灶）：《你说话是什么味儿》的 ❌/✅ 示范句"刮一道我得心疼死""配我那件燕麦色大衣"，**原样出现在 314 局对话**，尽管提示词明写"别照搬示范"。
- **性格反转失效**：317 是 C冷淡（设定"话不多"），实际是全场最话痨（连续 180/194/195 字）；319 A理性遇导购含糊回答本该不满意，却顺着夸。
- **话太多实锤**：Elena 单条普遍 150–220 字，固定"摸包旁白＋夸一句＋'不过'转折＋一长串担心＋追问"模板；并逮到复读 bug（314 开场整段输出两遍）。
- **结论**：随机变量"没起作用"的真因，**主因是提示词过度圈定（5 阶段标准问题清单 + 逐字示范 + 固定画像引力），次因才是随机内容简单**。产品只换颜色＝ADR 0003 边界（产品事实不进池）的必然，非 bug。

## 目标

把场景1 角色扮演链从百炼 Agent 切回 **业务后端组装全量提示词 + 直调百炼模型 API（qwen3.7-plus）** 运行（不经 Agent 应用编排），并把提示词重构为 **v9**：
1. 直调模型下上下文完整、"认历史"约束置顶，重复问/换问法显著减少；
2. 结构从 12 个扁平一级标题改为 **4 区分层 + 铁律置顶**，主次分明；
3. 单条发言变短、不再长篇解释；
4. 随机性格在**语气 / 话密 / 热络度 / 好不好说话**上拉开差异（风格层），不再被固定剧本压平。
随机性格 / 产品机制与百炼变量契约（`roleplay_personality` / `roleplay_product`）保持不变。

## 已定决策（brainstorm 拍板，作为本计划前提）

1. **调用方式**：角色扮演链弃用百炼 Agent 2.0，切「全量提示词 + 直调模型」。随机性格 / 产品机制保留；`${roleplay_personality}` / `${roleplay_product}` 变量占位契约不变（直调下由后端拼 prompt 时注入）。
2. **随机变量目标 = 风格层**（用户选 B，"先"做保守版）：大流程框架不变，让不同性格的语气 / 话密 / 热络度 / 好不好说话拉开；**不做**"性格驱动分叉"（问什么、买不买倾向都不同）——留作下一轮加码项。
3. **结构 = 4 区分层 + 铁律置顶**（详见下「prompt 新结构」）。
4. **5 阶段流程保留框架、删标准问题清单**：进店→需求→上手→掂量→临门顺序不动，但删掉每阶段写死的"你会问什么皮 / 纹路 / 容量"，降级为"心态怎么变 + 往下走的标志"。
5. **随机池本轮一并调整（与 v9 提示词配套）**：提示词大改后，灌进变量的随机池 content 必须同步适配，否则脱节。**性格卡按统一"风格旋钮"重写**（话密 / 热络防备 / 挑剔随和 / 推进快慢 / 情绪藏露），去具体台词、去行为分叉清单，让卡真正承载风格层差异；**产品卡适配 v9 措辞**（旁白指代 / 不报参数），守 ADR 0003 不加产品事实（产品"简单"是边界必然，本轮不破边界）。
6. **随机池 = 受保护产物，改动经授权**：`最新随机池内容.md`（+ 机制说明）改它＝改线上，执行沿用"开关代改"流程（临时放行 guard → 改 → 恢复 guard → JSON 校验 → code-reviewer），改后需在百炼侧重灌 `nstr_global_config`。

## 四个子任务设计

### ST1 切直调模型 + 提示词适配
- **后端 / 平台侧（用户操作）**：角色扮演从百炼 Agent 应用 → 改为**代码直调百炼模型 API（qwen3.7-plus）**；解绑知识库 / 关 `search_knowledgebase`（ADR 0004 待办）；后端组装全量提示词、注入两个随机变量。
- **提示词侧适配（本计划做）**：
  - 核对并清除正文中任何依赖"agent 工具 / 查资料 / 知识库"的残留措辞（ADR 0004 已重写《涉及产品事实》节，需通读确认无遗留）。
  - **强化"认历史"**：直调模型没有 Agent 记忆机制兜底，全靠提示词 + 全历史，故把"认历史不重来"提为铁律区第一条（见 ST2）。
  - 变量占位 `${roleplay_personality}` / `${roleplay_product}` 写法不变。

### ST2 结构分层重构（W1）
当前 12 个一级标题全平级，模型对所有内容等权重对待——这是过度圈定能压住性格的**结构性原因**。改为 4 区 + 铁律（见下「prompt 新结构」）：人设收薄、随机变量提到最显眼并提权、表演规则合并去重、心路瘦身、铁律集中显著标记。

### ST3 话太多治理（W2）
- 铁律新增："一条回复只说一个主要意思，别一口气又摸包又夸又担心又追问。"
- 删掉诱发长回答的饱满示范（见 ST4 / W4）。
- 话密随性子：冷淡就短、心动可多两句，但都别长篇——与风格层（ST4）协同。
- 复读 bug（W6）：加一条"别重复你刚说过的话"兜底；但根因多半在平台采样，提示词只能缓解，**验收时观察是否复发，不保证根治**。

### ST4 随机变量风格层（W3 + W4 + 随机池重写，含性格池）
**两条腿配套：提示词侧给变量提权 + 减负，随机池侧把 content 写成能穿透的风格差异。缺一边都穿不透。**

提示词侧：
- **W3 给性格变量提权**：在「你怎么演」区开头明确——"你这一局的说话方式，首先服从〔这一局的你〕的性子（话多话少 / 热络防备 / 挑剔随和），这条**优先于**下面所有通用说话建议"。把随机性格从被淹没的一句，提为表演的最高准则。
- **W4 删"示范变台词"**：《你说话是什么味儿》的 ❌/✅ 逐字示范 → 原则化描述（口语化、先反应后提问、带自己生活、有来有往），**删掉具体例句**（"刮一道心疼""燕麦色大衣"等高频复现句）。
- **开场分化**：阶段①不再写死"客气保持距离 / 随便看看"，改为按性子给不同进店姿态（冷淡才往后躲、心动主动凑近、理性直接说要看什么、比价边看边惦记别家）。流程没变，开场表现分化——风格层在阶段①的落地。

随机池侧（受保护，经授权）：
- **性格卡（4 张）按统一"风格旋钮"重写**，差异落在风格层、不写死问题清单、不含具体台词：
  - 旋钮维度：①话密 ②姿态（热络 / 防备）③挑剔度 ④推进节奏（快 / 慢热）⑤情绪藏露 ⑥**最先在意（关注点）**——红队新增、治"问材质趋同"的关键，与提示词区四③咬合。
  - A 理性＝话适中有条理 / 平和带审视 / 挑剔（含糊就不满意）/ 谨慎 / 藏着怕被拿捏 / **最先在意耐不耐用·值不值·细节经不经推敲**；B 比价＝话适中 / 较真不服气 / 爱比较权衡（非固定比价问题）/ 要被说服才推进 / 半露 / **最先在意跟别家比·凭啥这价·做工差在哪**；C 冷淡＝话少 / 防备往后躲 / 随和但慢热 / 慢需空间 / 淡不外露 / **最先在意第一眼颜色感觉·被问才淡淡说**；D 心动＝话偏多 / 热络主动 / 随和好说话 / 看对眼就快 / 藏不住喜欢 / **最先在意好不好看·配不配自己·什么场合背**。
  - 去掉与新提示词冲突的写法：如 C 卡"先撂随便看看"（具体台词，与"开场按性子分化"冲突）。
- **产品卡（6 张）核对适配 v9 措辞**：与旁白机制、不报参数、〔本局对应款〕指代方式对齐；守 ADR 0003 不加产品事实。
- **机制说明同步**：`角色扮演随机机制说明.md` 若涉性格卡写法口径，一并核对。

## prompt 新结构

| 区 | 标题 | 收纳 / 改动 |
|---|------|-----------|
| 一 | **你是谁**（人设底色·收薄） | 保留 Elena 核心（爱包舍得花 / 有主见识货 / 来线下试背 / 要被真听见）；压缩"生活底子"长清单，尤其旧包发硬那段（数据显示反复复述），删易被原样照搬的例句 |
| 二 | **这一局的你** ★随机变量提到最显眼 | `${roleplay_personality}`（性子：话多话少 / 热络防备 / 挑剔随和）+ `${roleplay_product}`（看上的包）；点明性子定本局脾气、优先于通用建议。**性格卡 content 按这套风格旋钮重写、与本区配套**（见 ST4） |
| 三 | **你怎么演**（合并：说话味儿＋一来一回＋旁白机制·去重删例句） | 开头给性格提权（W3）；说话味儿留原则删例句（W4）；一来一回（认历史、别重复问 / 别重做动作）；旁白机制（v8.5 版，克制原则不变） |
| 四 | **你这趟的心路**（5 阶段瘦身） | 保留进店→需求→上手→掂量→临门顺序 + 只进不退；每阶段只写"心态怎么变 + 往下走标志"，删标准问题清单；阶段①开场按性子分化 |
| ★ | **铁律**（最高优先级·集中显著标记） | ①认历史不重来（置顶，ST1 强化）②只进不退 / 没接住别清零 ③只说顾客的话 / 别认 AI ④一条只说一个意思（ST3 新增）⑤别报参数 / 事实靠问导购（ADR 0004）⑥听岔别纠正 / 整句没接住请重说或拎回上一问（v8.5）⑦别输出排版符号 / 少用省略号 ⑧结束机制（`<?END_CHAT>` 两段式 + 提前结束） |

## 步骤

> 有序、每步可独立提交。

1. **新建 v9 草案骨架**：创建 `草案/agent提示词_v9_直调重构_草案.md`，先搭 4 区 + 铁律的空骨架与顶部设计说明（标注 v9 顺位、继承 v8.5、切直调模型）。
2. **写区一 / 区二**：从 v8.5 草案迁《你是谁》（收薄）、《这一局的你》（随机变量提到区二、加提权与性子维度说明）。
3. **写区三**：合并《说话味儿》＋《一来一回》＋《旁白机制》，开头加性格提权句，删 ❌/✅ 逐字示范改原则化（W3 / W4）。
4. **写区四**：5 阶段瘦身——删每阶段标准问题清单、降为心路，阶段①开场按性子分化。
5. **写铁律区**：集中 8 条硬约束，认历史置顶，新增"一条只说一个意思"与复读兜底，迁结束机制。
6. **ST1 适配通读**：全文搜查 agent / 查资料 / 知识库残留措辞并清除；确认变量占位写法；确认无依赖 Agent 特性的表述。
7. **重写随机池（受保护，经授权）**：沿用"开关代改"临时放行 guard；性格卡 4 张按风格旋钮重写、产品卡 6 张核对适配 v9 措辞、机制说明同步；改完恢复 guard，校验 JSON 可解析、`{name,content,enabled}` 字段完整、契约变量未动。
8. **写 / 补对抗自测清单**：新增风格层用例（C冷淡话少易走 / D心动热络快推进 / 开场不再统一随便看看 / A理性遇含糊会挑剔），及话密、不复述示范句、单条不超长的检查项。
9. **卫生检查 + 回写台账 + 收尾复核**：`bash scripts/verify-local.sh` 硬拦项全过；回写 `0_项目总览.md`（角色扮演链状态 / 决策日志 / 变更记录）；`/rig:adr` 记"切回直调模型"跨域决策；改了受保护随机池，收尾跑 `code-reviewer` 复核。
