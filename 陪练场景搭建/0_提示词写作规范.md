## 一、三条核心设计原则

### 1. 旁白只给"客观可见"，绝不泄露内心
陪练是纯文字模拟，顾客台词后用全角括号旁白 `（……）` 外显"文字承载不了、却影响真实感与流程"的信息——但**只给客观外显**：体感动作（伸手拿起、掂重量）、产品锚定、可见信号、对 SA 动作的反馈。
**铁律**：旁白**绝不泄露内心 / 动机 / 过招骨架**（"她其实觉得太贵但不想说" ❌ 不进旁白）——内心留到**会话后复盘**揭示，不破坏"SA 不可见随机信息"原则。
> 来源：`0_项目总览.md` 角色扮演链旁白机制。

### 2. 事实层 RAG 锁死 vs 表达层口语放开（解"僵硬"的总钥匙）
解决"建议 / 话术僵硬" = **事实层用 RAG 锁死防幻觉 + 表达层放开口语情绪**。配套：KB 应存"口语化标准话术"而非纯书面参数。这是评估链与角色扮演链共用的全局设计原则。
> 来源：`0_项目总览.md` 跨链总结论（升为全局设计原则）。

### 3. 画像带情绪 / 立场演化，不写成静态长段
画像若写成静态长段、无情绪状态机 → 客户平铺复述设定、不像真人。**画像要带随对话演化的情绪 / 立场**（阶段情绪曲线），让顾客像真人会犹豫、会被打动。
> 来源：`0_项目总览.md` 跨链总结论。

## 二、写作铁律（与 `docs/conventions/code.md` 同源，全文在此）

- **零编造**：缺失的价格 / SKU / 期限 / 尺寸标 `【待客户补充】` 或 `【以门店主数据为准】`，不凭印象推断。
- **专有术语原样保留**：END / FABS / BFA / Tabby / Expressive Luxury 等按原文，不意译、不合并相近缩写（FABS≠BFA）。
- **异议写"关切主题 + 潜台词"口语短句**（10–25 字），不写完整台词（防机械复读）。
- **版本号语义**：改了"行为"才升版本号，只收瑕疵走小数点；旧结论删除线作废、不静默覆盖。
- **百炼变量契约**：`roleplay_personality` / `roleplay_product`，对齐后端 `NstrRoleplayCard`。

## 三、防护与活度（精简版心法）

- 身份防护 + 行为边界保留**精简版**（核心铁律去重）；过度"禁止"转"应该怎样"。
- 正向人设："你是有血有肉、会犹豫会被打动的真实顾客"，给角色活度留空间。
- 单一焦点**软化**为风格倾向（鼓励一次聚焦一个关切、允许自然追问 / 带情绪 / 带铺垫），废除"机械数问号自检"。
> 来源：`0_项目总览.md` 防护模块繁简度决策。
