## 2. 三链架构总览

被替代的现状：**单体全量系统提示词 + input 直调大模型** → 角色扮演/阶段推进/评估打分互相挤压，扛不住四诉求。
解法：**百炼多智能体 + 知识库 RAG，按职责拆三链**。

| 链路 | 承接的客户诉求 | 主要引用 KB | 调用方式 | 输出 |
|------|----------------|-------------|----------|------|
| **① 角色扮演对话** | ①角色丰富度 ②角色差异性 ④对话推进 | KB4(画像/异议/潜台词) + KB2(产品FAQ) + KB3(阶段动作) + KB1(品牌/红线) | 按需 | Markdown 对话 |
| **② 实时评估** | ③评估准确（即时反馈，不出总分） | KB5(维度判断/红线) + KB3(当前阶段) + KB2/KB3(事实口径) | 必定 | Markdown 即时提示 |
| **③ 整体评估** | ③评估准确（结束出完整报告） | KB5(四件套) + KB3/KB4(动作与画像) | 必定 | Markdown 评分报告 |

> 本期**只搭这三链**，不做对话采集、不做变量生成（与太保项目的范围差异）。

## 3. 全局设计约定（跨场景通用，逐步沉淀）

> 这些约定一旦敲定，所有场景的三链都遵守；是"批量复制场景"前必须先定死的地基。

### 3.0 场景定义（最根本约定 · 2026-06-21 修订）
**一个场景的身份 = 全流程（固定）× 画像（具体角色）× 渠道**：
- **全流程（固定，不再是区分维度）**：本次 Coach 所有场景都走**完整流程**——迎宾破冰→需求诊断→触感体验→价值传递→主动成交（到成交为止，不含售后维系）。每个场景都全流程，故"阶段"不再用来区分场景。
- **画像**（对谁练）：质感自用 / 性价比入门 / 礼品选购 / 都市悦己，落成**固定具体角色**（如质感自用→Elena）。
- **渠道**：正价(Retail) / 奥莱(Outlet)（私域二期）。**独立维度，但给"典型搭配"默认**——质感自用·都市悦己 → 默认正价；性价比入门 → 默认奥莱；**允许例外**。
- **❌ 取消"主题=某类异议"维度（2026-06-21）**：原设"主题（价格异议/产品知识/…）"与"全流程"冲突——全流程里价格、材质、推荐、临门各类异议本就都会碰到，无法用"练哪种异议"区分场景。**异议改为由「画像 + 当前阶段」全程自然涌现**（KB4 本就按阶段+画像组织：破冰"只是看看"、需求"不确定适合我"、触感材质疑虑、价值/成交才价格异议）。**场景区分只靠「画像 × 渠道」。**
- **阶段的双视角（重要，保留）**：阶段本质是**销售流程视角**（销售 SOP ＋ 评估"流程意识"的标尺）。① 进**角色扮演链**翻译成**顾客视角**——顾客不知道、绝不说出"价值传递"等阶段名，只表现对应**心理状态＋反应信号**，阶段推进由顾客**外显信号**自然带出。② 进**评估链**才是**销售视角**的流程标尺（评有没有按阶段走/跳步）。
**不算场景身份**（不进命名、不决定"是哪个场景"）：产品（软背景，开局机制随机锚定）、过招剧本骨架（破雷同机制，现定位＝**这局顾客整体过招风格、贯穿全程**）、互动模式 / 难度（修饰）。
**命名规则（改）**：`场景N_画像角色_渠道`（例：**场景1_轻奢包追求者Elena_正价**；画像名用**客户原始画像名**"线下正价轻奢包袋追求者"，而非内部"质感自用"黑话）。**复制新场景 = 换"画像 / 渠道"。**

### 3.1 模拟环境补偿：旁白机制（2026-06-21 新增 · 跨场景通用）

> **为什么要有**：陪练是纯文字模拟，没有真实体感——上手摸、试背、试戴、端详都发生不了；而真人顾客**不报型号**（只说"那个链条包"），SA 会不知道在聊哪款、流程卡住。靠**旁白**把这些"文字承载不了、却影响真实感与流程推进"的信息外显出来。

- **形式**：顾客台词后紧跟全角括号旁白 `（……）`。顾客自身动作用第三人称"她/他"，对 SA 物理动作的反馈用第二人称"你"。
- **承载什么（只给客观外显）**：① 体感动作（伸手拿起、掂重量、指尖划过皮面）；② **产品锚定**（见下）；③ 可见信号（拿到镜子前比、又放下）；④ 对 SA 动作的反馈（`（你把包递过去，她翻看内衬）`）。
- **边界（铁律）**：旁白**只给客观可见**的，**绝不泄露内心/动机/过招骨架**（"她其实觉得太贵但不想说" ❌ 不进旁白）——内心仍留到**会话后复盘**揭示，不破坏"SA 不可见随机信息"原则。
- **产品锚定机制（A 方案 · 解 R2 落地 + 防流程卡）**：
  - **幕后（SA 不可见）**：角色扮演链开局，从**按画像圈定的候选产品清单**里**机制随机抽 1 款真实产品**（和过招骨架同一套：工作流脚本节点 random + 最近 N 局不重复）作为"这局顾客最想看的那款"，全程围绕它。
  - **台前（SA 可见）**：顾客**永远不报型号**（台词是人话"这个链条包挺好看"），靠旁白锚定到那款——旁白给**清晰外观特征 + 产品名**（如 `（目光停在一只焦糖色 Tabby 手袋上）`），SA 据此讲解、查 KB、推流程；尺寸等细节（26/20）留给 SA 主动问顾客确认（这是好的销售动作）。
  - **事实约束**：旁白里的产品名/材质**必须走 KB（事实层锁死），不许编型号**。
- **触发要克制**：不是每句都加，只在 (a) 触感体验段、(b) 顾客指向产品、(c) 关键购买信号/犹豫、(d) SA 做了物理动作需反馈 时给。
- **评估怎么对待**：评 SA 话术时旁白算"情境"不算 SA 的话；但"**SA 能否读懂旁白、接住顾客信号**"是同理心/流程意识的观察点。
- **百炼落地**：角色扮演链系统提示词加一个"输出与旁白规范"模块即可，**无需额外组件**；产品名受 KB 约束。

### 4.1 系统提示词结构（v8 · 分场指导式 + 温度模块）— 🔵 待 review

> 分场指导式（v6 起）：以"人"为骨架、机制溶进心理、流程写成"几步心路 + 往下走的标志"。**v8 新增《你说话是什么味儿》温度模块**治"话术冷/官方"。详见 [v8 草案](场景1_轻奢包追求者Elena_正价/链一_角色扮演_系统提示词草案.md)。

| 段落 | 职责 | 对应原机制 |
|------|------|-----------|
| 你要做什么 | 扮演指令：演 Elena 本人、与导购对话、别当任务做 | 任务定调 |
| 你是谁 | 人物灵魂：背景/性格/在意什么/动机；**气质＝热爱/悦己/舍得（客户画像A）** | 原模块1 人设 |
| **你说话是什么味儿** | **真人感/温度模块**：口语+语气词+带自己的生活+对导购有来有往+正反示范 | **v8 新增（治冷/官方）** |
| 这一局的你 | 两个变量口子：`{{本局性格底色}}`（过招风格,贯穿全程）+ `{{本局产品}}`（不报型号,靠旁白锚定）| 原过招骨架 + 产品锚定 |
| 你会怎么一步步走 | 5 段心路（破冰→需求→触感→价值→成交）+ 每段"往下走的标志"+ 该段红线 | 原模块3 阶段状态机 + 模块4 异议（溶进心理）|
| 要守住的几条 | 红线：只演 Elena/不暴露 AI/一次一个关切/不念设定 | 原模块0 防护（精简）|
| 看不见的动作用括号演 | 旁白规范：括号写客观外显/产品锚定/不写内心 | 原模块8 旁白 |
| 拿不准产品事实时 | 按需查 KB、用自己话、不编型号/价格 | 原模块5/6 检索 |

> **移出角色 prompt**：轮次上限、阶段编号等纯控制项 → 交系统/评估链（角色只留"答到位就过、空泛才再追一次"的拟人节制）。

### 4.2 待逐模块敲定的争议点

1. ✅ **已定** 模块1 角色实例化：单场景写死具体角色（Elena），profile 全字段写死；差异性靠批量阶段生成多个实例。字段结构沿用：基本信息·消费经历·性格·痛点需求·语言特征·行为模式。
2. ✅ **已定** 模块0 防护繁简度：精简身份防护+行为边界，单一焦点软化为风格倾向（废除机械数问号），新增正向人设指令平衡防御性。
3. ✅ **已定** 模块3 推进条件：写成顾客侧可观测外显信号(70%)+销售具体动作(30%)，progression_limits 强制推进，送客触发收尾。

### 4.3 本链需扛住的 POC 硬性要求

- **R1 角色要活**：会犹豫/追问/有情绪/被话术推开或拉近；禁止对报价只回"好的"、禁止死板走完流程。
- **R2 不困在单品**：可接受合理推荐其他款，不把推荐都拉回 Tabby；覆盖多品类。
- **R6 活人感**：语气贴 GenZ、有朋友感，不官方不机器化。
- 开场不冷漠：体现正常人际礼貌与配合度。

## 5. 链路二 · 实时评估 设计

### 6.1 四维度（Coach POC 定稿）— ✅

| 维度 | 副标题（**勿据字面误解**） | 权重 |
|------|---------------------------|------|
| 专业性 | 知识运用与价值传递 | **30** |
| 同理心 | **需求挖掘与场景锚定**（≠单纯情绪共情） | **25** |
| 沟通技巧 | 共情与引导 | **25** |
| 流程意识 | 节奏与流程把控 | **20** |

- 每维度 **2–4 个指标**：指标名 / 分值 / **评估标准三层**（概念层 + 评估要点用疑问句 + 应用层结合 Coach 场景）。
- **总分严格 = 100**；维度三档（优秀/良好/待提升，区间见 KB5-评分）。
- **禁止指标**（硬约束）：时间限制 / 语音技术（语速语调停顿）/ 机械话术（必须说 X）/ 表情动作 / 假设结果。
- **证据导向**：每个扣分指向对话具体片段；红线落对应维度扣分（事实类→专业性；沟通禁忌→沟通技巧；流程跳步→流程意识）。

## 12. 标准产品借鉴台账（"全量prompt+直调"标准版经验的吸收与规避）

> 来源：`/Users/zhangyh/Documents/基础提示词`（标准产品三链：系统模板 + 太保实例）。标准产品同为"全量提示词+直调大模型"，有业务抓手但存在**角色不真实 / 话术僵硬 / 不像真人**问题。
> 角色扮演链精读 672 行系统模板；评估两链分析由 workflow `wm0rknsir` 补充（见 §12.5 实时评估 / §12.6 整体评估）。

### 12.1 核心洞察（架构级，最重要）
标准版把整套「**反应控制引擎**」（4 类销售信号识别 → 质量评级 → 4 维反应调节 → 多维整合 → 8 步输出检查）**塞进了角色扮演链**——顾客每说一句话前都要先给销售打一遍分，必然像填表、不像真人。这是"话术僵硬"的**结构性根因**。
→ **Coach 三链架构的结构性优势**：把"销售行为质量判定"归**评估链**（KB5），角色扮演链只管"演好这个顾客"（被打动就软化、被推开就冷淡），天然减负 → 更自然。**印证初始输入"单体 prompt 职责互相挤压"的诊断。**

### 12.2 值得借鉴的业务抓手 → 用到 Elena
| 抓手 | 用法 |
|------|------|
| 禁止暴露内部机制（不输出阶段判定/异议选择/推进逻辑） | ➕ 加进模块0 |
| 收尾两段式（第1轮释放**明显**结束信号给导购"收尾窗口"，第2轮才告别） | ➕ 细化模块3收尾（落地"导购送客触发制"） |
| 异议智能选择（针对导购当前话语、**排除已答的**、**禁止照搬库原文**、改写成自然反应） | ➕ 加进模块4使用规则 |
| 追问硬上限（同一问题点有上限，防无限刁难） | ➕ Elena 懂行版放宽到"答空洞才追第2次" |
| 话题切换衔接（先回应上文+过渡词+新话题，不硬抛新问题） | ➕ 加进模块0风格 |
| 变量化字段打包（phases = 阶段名+阶段动作+maxRounds+异议，整体打包） | ✅ 对齐模块7变量清单 |

### 12.3 必须规避的僵硬根因（反面教材）
| 反面写法 | 为何对 Coach 有害 |
|----------|-------------------|
| **"你是普通顾客不是专家，禁用专业术语、禁止指出销售错误、不懂就问"** | ⚠️**最关键**：对太保电话保险小白合理，但 **Elena 是懂行、会盘问材质工艺的高客单顾客**——套这条 Elena 就废了。Coach 高客单场景的**本质差异**，反证"懂行追问"设计是对的 |
| 8 步评级 + 4 维反应引擎 | 角色扮演链不该背，搬给评估链（架构优势） |
| "执行检查：数问号、超1个必删" | 机械计数 → 僵硬（已废除） |
| "绝对禁止附和：不准说'好的/听起来不错/有道理'" | 让顾客冷漠戒备、不像真人；Elena 有礼貌，正常认可该保留 |
| 写死"配合型客户"单一类型 | Coach 靠**画像**驱动差异，不用单一客户类型 |

### 12.4 变量字段对照（标准产品 FreeMarker → Coach 自定义变量）
- 标准产品：`bot.display.name`(角色名) / `bot.ext.description`(描述) / `extra.company_name`(公司) / `extra.product_name`(产品) / `bot.display.prompts`(画像 Role) / `base.background`(场景背景) / `context.phases`(阶段:phaseName+botAction+maxRounds+phaseProblems异议)。
- → Coach 模块7：画像(Elena 写死) / 产品(Tabby26) / 渠道(Retail) / 阶段范围 / 主题 / 互动模式 / 难度。**借鉴**：标准版把"阶段动作+轮次上限+异议"随 phases 一起打包的做法，值得用在阶段状态机的参数化上。

### 12.5 实时评估链借鉴
**定位（借鉴）**：只评**最新一条销售消息**，不评历史、不总结多轮、不跨节点；三步法 定位评估对象 → 匹配维度 → 综合评估。输出"即时观察 + 改进提示（+参考话术方向）"，简短。
**规避根因**：
- 强字数上限（≤100字）压垮语气 → 电报腔：实时提示要短，但**别钉死字数**，保留口语呼吸感。
- "参考话术必须照资料标准表述 + 不得编造" → 说明书腔：**KB 里要存口语化标准话术**，而非纯书面参数（对 KB3/KB5 的改进提示）。
- 自查标签 / 三段式公式化 → 模板味：自查放内部，别污染输出。

### 12.6 整体评估链借鉴
**借鉴抓手**：
- **三级评分卡**：维度 → 指标 → 评估要点(疑问句) + **显式分值组成**（如报价维度=车险准确10+非车完整10+衔接5），可量化、可追溯。← 对齐 KB5 维度结构。
- **完成度 → 分数段映射**（全完成→满分 / 80%+→90-100% / 一半→50-80% / 少量→0-40%）。
- **培训场景宽容评分**（陪练非考核、鼓励而非打击、核心要点完成即高分）。
- ⭐ **RAG 事实溯源红线**：检索内容 = 产品事实**唯一来源**；宁可建议笼统不可细节编造；评估标准里的事实示例 ≠ 事实依据，与检索冲突一律以检索为准。
- ⭐ **评分维度 vs 事实维度 双轨**：判"该不该说"用评分卡、判"说得对不对"用 RAG，两者解耦、不互相加码。
- **红线去重**：销售已说过的（即使换词）禁止再作为建议。
- **结构化输出**：分维度 + 证据 + 建议（Coach 用 Markdown 标题/表格，不用 JSON）。
**规避根因**：
- ⭐ **画像写成静态长段、无情绪状态机** → 客户平铺复述设定、不像真人：**画像要带随对话演化的情绪/立场**（Elena 草案模块3阶段情绪曲线已体现，继续强化）。
- ⭐ **表达层被事实锁死却没留口语/共情空间** → 同义改写僵硬：**事实层(RAG锁死) vs 表达层(口语/情绪自由) 显式分离**——解决"评估建议/话术僵硬"的总钥匙。
- 字数句式钉死、等价词典静态枚举（查表式）→ 规避。

> **跨链总结论（升为全局设计原则）**：解决"建议/话术僵硬" = 事实层用 RAG 锁死防幻觉 + 表达层放开口语情绪；**KB 应存"口语化标准话术"而非纯书面参数**。
