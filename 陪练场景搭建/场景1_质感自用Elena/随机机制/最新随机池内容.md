# 场景一 · 角色性格 / 产品 随机池

> single source of truth：改池先改这里，再灌到 `nstr_global_config`。
> 字段（与代码 `NstrRoleplayCard` 对齐）：`{name, content, enabled}`。`name` 仅内部识别、不注入；`content` 注入提示词变量；`enabled=false` 即下架。
> 性格池 config_key：`nstr.roleplay.personality-pool` → 注入 `${roleplay_personality}`（变量名以代码为准，非 persona）。
> 产品池 config_key：`nstr.roleplay.product-pool` → 注入 `${roleplay_product}`。
>
> **设计原则（2026-06-24 立 · 2026-06-25 细化）**
> - **产品池 = 知识库的"轻量索引"，不重复产品事实**。`content` 只写两样：① Elena 眼里这只包的外观口语；② 末尾一个〔本局对应款〕锚点。皮质/五金/保养/价位/精确尺寸等**产品事实仍不写在这**（守单一来源），AI 被问到去查知识库（Tabby/Brooklyn 系列库）。
> - **（2026-06-25 细化）款式身份可由"旁白"点明**：〔本局对应款〕不再是"绝不说出口的内部锚点"，而是这只包的款式身份——AI 可在**旁白**（提示词《旁白机制》定义的括号呈现层）里点明在看哪款，让学员锚定该介绍什么；但顾客**对话口语**仍用"那只焦糖棕的""那个黑色方包"自然指代（真人感）。**放开的只是"哪款包"的指认、不是产品事实细节**，参数仍走知识库。
> - 产品只用知识库真实存在的款（Tabby 26/20、Brooklyn 28）+ 真实颜色，调性贴画像审美（大地色、经典耐看、不甜不张扬）。
> - **性格池写"这一局怎么反应"的行为卡**，对齐 Bella 奥莱场景 2026-07-01 调整：固定包含说话气质 / 最先关切 / 证据阈值 / 追问策略 / 被打动信号 / 被推开信号。画像底色交给提示词《你是谁》，卡里不重复画像、不写具体台词、不写固定流程脚本；每张卡必须带追问后的出口动作，防止性格特征被模型放大成连续盘问。
>
> **变更记录**
> - **（2026-06-26 扩充 · 对齐业务产品列表 `coach产品-tabby.xlsx` 44 款）** 产品池 6 → 15 款：新增款全部取自 Tabby 真实 SKU、贴画像（大地色 / 经典 / 不张扬），张扬款（水晶 / 扎染 / 珠饰 / 印花 / 丹宁拼布 / 西部 / 纽约主题等）不入池。**卡名 `name` 末段带 SKU（产品编号）便于追溯主数据**——`name` 仅内部识别、不注入，故不影响 Elena 对话。Brooklyn28×2 与雾蓝灰季节限定×1 无 Excel 编号、原样保留。产品事实（材质 / 尺寸 / 价格）仍只在知识库（`KB2-正价-Tabby系列.md` 逐款明细），不写入池。
> - **（2026-07-01 调整 · 对齐 Bella v0.4 行为卡结构）** 性格池由旧版短描述（A-理性型 / B-比价型 / C-冷淡型 / D-心动型）升级为 4 张 Elena 行为卡（A-价值型 / B-比选型 / C-怕买错型 / D-心动型），结构与奥莱 Bella 一致，但关切替换为价格价值、质感耐用、搭配场合、打理售后、版型比例、容量通勤等门店顾客自然关切。
