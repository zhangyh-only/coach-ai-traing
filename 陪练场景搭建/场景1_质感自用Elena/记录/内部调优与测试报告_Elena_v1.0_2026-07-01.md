## 背景

奥莱 Bella 场景修复后，正价 Elena 场景需要同步同一套角色扮演稳定性原则，但不能带入奥莱画像里的“正品 / 折扣 / 奥莱渠道”底色。Elena 的核心顾虑应回到正价门店顾客更自然的判断逻辑：

- 这只包值不值正价，为什么选它而不是别家或另一款。
- 质感、耐用、打理和售后是否支撑长期使用。
- 上身比例、颜色、通勤和聚会场景是否适合自己。
- 容量、重量、肩带、开合内里等体验点是否足够舒服。

## 调整范围

### 提示词

文件：`草案/角色扮演链提示词_Elena_v1.0_草案.md`

线上最新 remark：`Elena v1.0 deal end hard gate`

关键调整：

- 采用“内部材料 / 人设 / 性格行为卡解释 / 说话与旁白 / 状态账本 / 同类关切消耗 / 产品事实边界 / 收尾机制”的结构。
- 模型可见正文避免高频出现“正价”字样，统一改成“Coach 门店 / 这个价位 / 值不值 / 为什么选这只”，避免诱发 Elena 反问“是不是正价”或把价格问题放大。
- 将 `<?STATE>` 固定为 `关切 / 动作 / 表态` 三字段，并要求每一轮第一行必须输出。
- 将 Elena 的同类关切按正价场景重组为：价格价值、质感耐用、搭配场合、打理售后、版型比例、容量通勤、重量舒适、肩带背法、开合内里、退换。
- 明确“材质名、清洁难度、日常保养、官方护理服务、售后承接”属于相邻问题，导购给过任意实在说法后不再拆成多轮重问。
- 产品锚点增加颜色约束：口头颜色必须沿用本局产品卡或同一上位色，不能把暖棕、太妃棕、鞍棕等相近色互相改写。
- 旁白规则收紧：括号只允许导购看得见的动作、视线和拿包动作，不能写心理活动。
- 收尾规则补强：成交、再想想、不买都走两段式；Elena 先表态，导购送客后，下一句自然告别并加 `<?END_CHAT>`。
- 追加“已聊同类关切提醒”硬门闸：只要已聊历史出现同类提醒，该组就视为关闭，不能换场景重新提问。
- 追加 `<?STATE>` 与台词一致性例子：链条太重、颜色太跳、麂皮打理、价值判断等句子若写成 `关切=无`，视为错误。
- 成交送客补强：上一轮已“要买 / 拿新的检查 / 包起来”，导购说“已经包好了 / 谢谢您”时，必须在自然告别后加 `<?END_CHAT>`。
