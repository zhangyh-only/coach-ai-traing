## 0. 这个冒烟版要回答 6 个问题（验证目标）

| # | 要验证的机制 | 通过的标准 |
|---|-------------|-----------|
| V1 | **Elena 活不活、像不像真人** | 会犹豫/追问/被打动/被惹烦；对报价不只回"好的"；不死板走流程 |
| V2 | **防护到位** | 不暴露自己是 AI、不扮演销售、不替导购说话 |
| V3 | **性格底色管不管用（贯穿全程）** | 手动给一种性子（如"理性盘问"），Elena **从破冰到成交全程**带着这股劲；换一种性子（如"防御冷淡"），整局味道明显变 |
| V4 | **防 AI 念稿** | 开局**不会**把"你今天的性子"这类内部设定直接说出来 |
| V5 | **知识库 RAG 能检索、答得准** | 被问材质时会去查知识库，答得准、带来源角标；价格这类没有的，不乱编 |
| **V6** | **旁白 + 产品锚定** | ① 顾客**不报型号**，但靠括号旁白让你知道她在看哪款；② 旁白**只写客观外显**（动作/神态/目光），**不泄露内心**（不出现"她觉得贵"这类）；③ 旁白里产品名/材质是 **KB 真实**的、不编型号；④ 触感段有摸/试背的体感旁白；⑤ 旁白**克制**，不是每句都加 |

> 📌 **思考过程（Thinking）不用担心**：Qwen3 带推理，调试窗会显示"阶段①/防备状态"这类内部推理，但这**仅调试时可见，上线由业务系统隐藏**，SA 看不到——机制不外泄，无需额外验证。

## 1. 冒烟版范围裁剪（先做 / 先不做）

- ✅ **先做**：1 个智能体应用（Agent 2.0）+ 挂知识库 + **用 v6 冒烟即用版系统提示词**（两个口子已填死：性格底色＝理性盘问、产品＝焦糖 Tabby）。旁白规范已含在 v6 正文《看不见的动作，用括号演出来》。
- ⏸ **先不做**（等冒烟通过再上）：业务代码自动随机抽性子/产品、记忆库、去重、评估难度找平、自定义变量动态赋值。

## 2. 搭建步骤

### Step 1 · 建知识库，传 KB，打标签
- 新建一个知识库（文档型/标准版）。
- 冒烟版先传**场景1 用得到的几篇**即可：`KB4-画像-质感自用型`、`KB4-异议-阶段分布索引`、`KB4-潜台词与决策焦虑索引`、`KB2-正价-皮革与材质知识`、以及**本局产品所在那篇**（如 Tabby 系列）、`KB3-方法-皮革材质异议应对话术`。
- 给每篇打**标签**：库编号（KB2/KB3/KB4）+ 分类（画像/异议/产品/方法）——后面检索靠它缩范围。

### Step 2 · 建智能体应用（Agent 2.0）
- 新建应用，选**智能体应用（Agent 2.0）**。
- 模型选 **Qwen3.7-Plus**（实测优于 qwen-max；qwen 迭代快，以平台最新型号实测为准）。

### Step 3 · 挂知识库
- 把知识库挂到这个应用上（挂上后 Agent 2.0 会自带一个 `search_knowledgebase` 检索工具）。
- **不用找"按需"开关**：Agent 2.0 里知识库是"工具"，模型按 ReAct 自己决定要不要查、查什么——这天然就是"按需"。提示词里 v6《拿不准产品的事实时》已引导它"需要核对才查"。
- 「预解析文件」的「全文引用 / 切片检索」是**另一回事**（针对对话里上传的文件，不是知识库），冒烟阶段保持默认即可，预解析开关也可以关掉。
- （可选）若检索召回到不相关库（如评估库 KB5），再去知识库设置里用标签限定 KB2/KB4。

### Step 4 · 填系统提示词（关键）
- 直接用 [v6 冒烟即用版](../归档/冒烟即用_系统提示词_骨架A_焦糖Tabby.md)：把它 `▼▼▼ … ▲▲▲` 之间**整段**复制，粘进系统提示词框即可——两个口子（"你今天的性子" / "本局产品"）已填死，**无需手动改占位**。
- **换性子重测**时：把"这一局的你 · 你今天的性子"那段换成别的（如**防御冷淡**：全程话少、淡淡的，要先被破冰建立信任才肯多聊、后段才进价格），其余不动。

### Step 5 · 其余配置
- 短期记忆：**≥20 轮**，要够覆盖整段全流程（"走到第几步/聊过啥/看哪只包"靠对话历史，记忆不够会乱）。
- 多样性/温度（若可调）：**略偏高**（更活、跨局更不雷同，但别太高免得跑题）。
- ReAct 最大轮次：**10–15**。
- 打开**回复展示来源**（方便看它到底查没查知识库、查的哪篇）。

### Step 6 · 进对话调试
- 用应用自带的对话调试窗口，**你扮演导购**和 Elena 对练，**从进店迎宾开始走完整流程**。

## 3. 冒烟测试怎么测（你扮导购，从迎宾破冰走到成交）

> 全流程跑，重点看每一步的现象。括号里是该步要盯的验证项。

1. **迎宾破冰**：你说"您好，欢迎光临，随便看看，有喜欢的我帮您拿～"
   - 看 V1（Elena 像不像真人进店）、V4（会不会把"我这局要理性盘问"念出来）、**V6**（旁白有没有把她在看的"焦糖色 Tabby"指出来，且她**没报型号**）、V3（这股较真劲儿在不在）。
2. **给空间/搭话（破冰→需求）**：你别贴身，问"您平时喜欢什么样的包？什么场合用得多？"
   - 看她透不透露需求、阶段往不往前走；V3（理性、有条理）。
3. **触感体验**：你说"这只您可以上手摸摸、试背一下。"
   - 看 **V6**（有没有摸皮/试背的**体感旁白**，如"她伸手摸了摸袋身"）、V3（盘问材质）、V5（材质问题会不会查 KB、带来源、答得准）。
4. **空洞回答**：你回"就是大牌嘛，质量好。"
   - 看她会不会**不买账、继续追问**（被推开信号）。
5. **专业回答 + 报价**：你讲"进口头层皮、LWG 认证、独家复鞣、一次裁一个包"，并把价格夹在价值里说。
   - 看 V3（价值段她按"理性盘问"追溢价/耐用/保养）、V1（对报价不只回"好的"）、V5（你故意讲错"完全防水"，她该起疑）。
6. **临门成交**：她出现购买信号（旁白如"拿着包走到镜子前比了比"）后，你读信号、坚定请她买 + 复述确认。
   - 看 **V6**（购买信号有没有用旁白体现）、成交/退回是否合理、收尾**送客触发制**（她先放倾向信号、等你收尾才告别，不抢先结束）。
7. **换性子重测**：把 v6 即用版"你今天的性子"那段换成"防御冷淡"（全程话少、淡淡的、要先被破冰建立信任才肯多聊、后段才进价格），重开一局走全流程，看 V3（**整局味道是不是明显变冷、和理性盘问那局不一样**）。

> **额外盯一眼 V6 的边界**：整个过程中，旁白**不该**出现"她心里觉得贵""她其实很想要"这类**内心戏**（那是泄露内心，应留到复盘）；旁白也**不该每句都加**（只在体感/指产品/关键信号时出现）。

## 5. 额外关键验证：自定义变量能不能"动态塞"（决定随机机制怎么落）

这步决定后面"自动随机抽性子/产品"用哪条路：
- 在系统提示词里把两个口子做成自定义变量（`本局性格底色`、`本局产品`），用 `/` 引用。
- 试试：**能不能在对话调用 / API 请求时给它们传不同的值**（而不是只能在控制台填死默认值）。
- **结论**：
  - 若**能动态传值** → 后续随机抽性子/产品可走"变量赋值"这条干净路（业务代码调用时传值）。
  - 若**只能填死默认值** → 走"业务侧把随机选好的那段性格底色直接拼进系统提示词文本"这条路（全量注入），不赌变量。
