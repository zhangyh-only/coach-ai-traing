# 测试诊断：Bella 材质打理复发

日期：2026-07-01

范围：场景2「奥莱私域轻奢 Bella」角色扮演链 v0.4 草案 + 后端 `rolePlayState` 历史账本装配。

## 现象

测试局里，Bella 先问这只棕色包是不是麂皮；导购说明是绒面、麂皮质感的皮料，并提示避水避油。随后 Bella 继续问这种绒面平时是不是难弄干净；导购又解释清洁注意。再之后 Bella 问门店或官方有没有护理服务；导购给出护理服务说法。

到这里，「材质/清洁/护理」已经聊过一组完整关切。但几轮后 Bella 又回到“摸起来是不是麂皮、这种材质好不好打理、会不会显旧”的同类问题上。

这不是正常谨慎，而是同一底层关切换皮重抛。

## 根因

1. 提示词虽然有 `打理耐用` 分组，但原写法只列了浅色脏、刮花、磨花、抗造、保养，没有把材质名、绒面/麂皮、清洁难度、护理服务明确并到同一组。
2. 性格卡里 A/D 卡都有“转去问打理”的出口，C 卡也有“打理耐用”安全感顾虑；如果不加边界，模型容易把“问打理”理解成新的风险清单。
3. 后端 `NstrRoleplayCustomVariableHandler` 原先只把历史 `<?STATE ...>` 按顿号/逗号去重，不会做底层关切归并。模型看到的是原始历史碎片，不一定知道“麂皮/绒面/护理服务”已经消耗了同一条线。

## 已调整

1. `角色扮演链提示词_Bella_v0.4_草案.md`：
   - 写 `<?STATE ...>` 时，材质名、绒面、清洁难度、护理服务都记到 `打理耐用`；
   - `同一个底层担忧` 规则里明确“材质/清洁/护理服务”属于同一组；
   - 产品事实边界补充：导购给过材质或日常护理说法后，不再反复核对学名或护理渠道。

2. `随机池_Bella性格卡_草案.md`：
   - A-精算型：打理只在被触发或未聊过时问一次；
   - C-怕踩坑型：护理服务也算打理同组接住，不把打理拆成材质、清洁、护理服务连续问；
   - D-心动型：只转向未聊过的打理，不把保养补成清单。

3. 后端 `NstrRoleplayCustomVariableHandler`：
   - 结构化拆 `关切/动作/表态`；
   - 追加“已聊同类关切提醒”，把正品/渠道、划算价格、搭配闲置、打理耐用、版型塌变形、容量、退换兜底等做底层归并；
   - 历史里销售输入也参与“同类关切是否已聊过”的归并扫描，但仍只从 Bella 自己的 `<?STATE ...>` 里抽动作/表态，避免把导购话术误写成顾客状态；
   - 单测覆盖 `打理耐用`、`正品/渠道安心`，以及“只有销售主动讲过材质/清洁/护理时也应闭合同类关切”的复发风险。

4. 后端最终 prompt 留痕：
   - 在角色扮演链调用模型前，把已经替换好 `rolePlayPersonality / rolePlayProduct / rolePlayState` 的最终 prompt 存入 Redis；
   - Redis key 按 `companyCode + recordId + loopIndex` 组织，保留 3 天；
   - 新增接口 `GET /ailearning/nstr/tool/getRoleplayPromptTrace?recordId=...&companyCode=...`；
   - 测试脚本新增 `prompt-trace <recordId>` 命令，后续诊断先看最终 prompt，再判断是 prompt 规则、随机池还是模型波动。

5. `角色扮演链_Bella_v0.3_回归清单.md`：
   - 新增 E16「材质打理闭合」。

## 回归标准

输入场景：

```text
Bella 已经问过材质名/绒面清洁/护理服务；
导购已经给出材质说明、清洁注意或护理服务说法；
已聊历史里出现 `关切=打理耐用` 或后端归并提醒。
```

通过示例：

```text
<?STATE 关切=无；动作=看镜子；表态=喜欢>

行，那打理这块我大概知道了。我再看看这个颜色平时通勤搭不搭。
```

或：

```text
<?STATE 关切=无；动作=无；表态=再想想>

那我心里有数了，这只我是喜欢的，就是回去还得想想会不会经常背。
```

失败示例：

- 继续问这种皮是不是麂皮。
- 继续问绒面是不是很难清洁。
- 继续问有没有官方护理服务。
- 继续问这种材质会不会显旧。

## 残留风险

如果模型没有稳定输出 `<?STATE 关切=打理耐用...>`，后端仍会尝试从历史对话里的材质、麂皮、绒面、清洁、护理、保养等词做归并。最稳的方式仍是提示词侧按固定 schema 写账本，后端侧同时扫描销售输入和顾客输出做二次兜底，两层一起生效。

- 重启后 3 个有效样本均确认：最终 prompt 已替换完成，无 `{rolePlayPersonality}` / `{rolePlayProduct}` / `{rolePlayState}` 残留占位。
