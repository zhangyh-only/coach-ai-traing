# 对抗审查：Bella v0.3 角色扮演链

审查对象：

- `草案/角色扮演链提示词_Bella_v0.3_草案.md`
- `草案/随机池_Bella性格卡_草案.md`

审查目标：不把“改完文案”当完成，而是用 Coach 既有坑模式 C1–C7 反向攻击提示词，检查它是否仍会诱发重复问、原地纠结、过早结束、出戏或性格穿不透。

## 结论

v0.3 主方向成立：状态账本、同类担忧消耗、奥莱三大顾虑、四张行为卡的结构已经能覆盖本轮测试暴露的主要问题。

本次对抗审查发现 5 个可优化点，已在草案中修复：

| 等级 | 问题 | 风险 | 修复 |
|---|---|---|---|
| P1 | 背景里写“奥莱不到一半价能买到正品经典款” | Bella 会提前替自己确认正品和划算，削弱后面对正品/价格的真实追问 | 改成“听朋友说奥莱不到一半价也可能淘到合适的 Coach” |
| P1 | 结束段只说“先别加标签” | 模型可能把 `<?STATE ...>` 也当成不能加，导致最后几轮状态账本断掉 | 改成“先别加 `<?END_CHAT>`”，并补充收尾轮仍要先写 `<?STATE ...>` |
| P1 | C-怕踩坑型写“会更沉默甚至结束” | 轻微催单就可能提前结束，不符合“早退只在太离谱时用” | 改成先“更沉默、说再想想”，只有连续被催或明显不被尊重才结束 |
| P2 | 口语长度规则没有排除内部标签 | 模型可能把 `<?STATE ...>` 算进一句话限制，导致真实口语过短或只剩标签 | 改成“真正说出口的话一两句；内部标签不算口语” |
| P2 | 退换兜底没有进入同类担忧表 | C 卡容易把“七天无理由/能不能换/买错怎么办”换皮反复问 | 增加“退换兜底”同类担忧组 |

## 对抗用例

| 用例 | 攻击输入 | 期望 Bella 行为 |
|---|---|---|
| T1 正品已聊过 | 已聊历史含 `关切=正品`，导购又说“放心，奥莱也是真的” | 不再换成“专柜一样吗/有没有小票”重新问；可冷一点、转向价格/搭配/决定 |
| T2 退换换皮 | 已聊历史含 `关切=退换`，导购答过七天无理由；下一轮导购问还担心什么 | 不再问“买错了能不能换/回去不喜欢怎么办”；可说“这个我心里有数了，我再看下上身” |
| T3 动作已发生 | 已聊历史含 `动作=试背、看镜子`，导购又说“你可以上身试试” | 不问“我能试背吗”；承认已经试过，在效果或决定上推进 |
| T4 价格空泛催单 | 导购连续说“很划算，今天最后一天”但不给依据 | 可追一次活动/折扣依据；追过仍空泛后冷下来或说再想想，不被硬催成交 |
| T5 END_CHAT 两段式 | Bella 已说“那就这只吧”，导购送客“我帮你包起来” | 回复先写 `<?STATE ...>`，自然告别句尾加 `<?END_CHAT>`；不重新看包 |
| T6 C 卡被催 | C-怕踩坑型，导购贴很近催她马上定 | 表现为拘谨、保留、再想想；只有连续被催或被冒犯才提前结束 |
| T7 D 卡被接住 | D-心动型，导购具体讲通勤/聚会搭配并说明活动 | 快速推进到试背、活动/售后或收尾，不把每套衣服都重新确认 |
| T8 工程措辞泄漏 | 扫描提示词正文 | 除三处模板替换位外，不出现“变量/占位/程序/注入/大模型/直调”等工程叙述 |

## 当前残留风险

- 这次是离线对抗审查和结构检查，能拦提示词自相矛盾、工程视角泄漏、状态账本断点；不能替代真实百炼模型多局稳定性测试。
- 随机池仍是草案，未灌入公共随机池 SSOT；上线前还需要按“场景2独立性格池 config_key”走受保护变更流程。
- `rolePlayState` 是否真的每轮稳定回填，最终取决于后端抽取 `<?STATE ...>` 的实现。提示词已经要求固定三项，但仍建议上线前用真实接口检查 2–3 局状态回填。
