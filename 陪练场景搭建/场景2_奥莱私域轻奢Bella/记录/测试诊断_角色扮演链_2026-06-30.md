# 测试诊断 · Bella 角色扮演链 · 2026-06-30

数据来源：`/Users/zhangyh/Downloads/对话记录(评估模型输入) (4).xlsx`

范围：场景2「奥莱私域轻奢 Bella」角色扮演链。当前运行前提按用户 2026-06-30 补充：角色扮演链不再使用百炼 Agent 应用，而是业务程序拼装全量提示词后直调百炼大模型；变量统一为 camelCase：`{rolePlayPersonality}` / `{rolePlayProduct}` / `{rolePlayState}`；项目同时存在正价 Elena 与奥莱 Bella 两个画像角色。

## 结论

这批问题的主因不是 Bella 人设本身不对，而是三件事叠加：

1. `{rolePlayState}` 被写成“绝对不可以提的事情”，但没有被明确写成“必须读取的内部历史清单”。模型容易把它当成禁区，而不是判重依据。
2. `<?STATE ...>` 的本轮记录不稳定，多轮明明问了关切或做了动作，却写“无”或不写标签，导致程序提取的历史状态天然缺项。
3. Bella 的三大奥莱顾虑（正品、划算、搭配）被种在画像底色里，但缺“消耗机制”。模型会把它们当作可以反复续写的话题，换个说法继续问。

## 逐项诊断

### P0 · 状态回环没有闭紧，导致“问过又问 / 做过又做”

【现象】
- 记录 356 第 8 轮已追问“软皮背久了没型 / 装满会不会太重”，第 9 轮又问“这皮子这么软，背久了会不会塌啊”。这是同一个“软皮会不会塌 / 变形”的底层担忧换皮重抛。
- 记录 359 第 3、4、5 轮连续围绕“焦糖色适不适合通勤/周末/衣柜”打转，其中第 5 轮 `<?STATE 无>`，但实际又在纠结搭配。
- 记录 355 第 13 轮继续追“几百到底是几百”，但这一轮没有写 `<?STATE ...>`，如果后端依赖标签提取，会漏掉这次具体价格追问。

【根因】
- 最新 prompt 顶部“你绝对不可以提的事情：{rolePlayState}”只强调不能说，没强调要读、要用、要按它判重。
- `<?STATE ...>` 的格式过自由，模型会漏记“搭配 / 价格 / 动作”，或误写“无”。

【落点】
角色扮演链 → 重要前提 / 认历史不重来 / `<?STATE>` 状态机制。

【改法】
- 把 `{rolePlayState}` 改成“内部只读历史备忘录”：必须读取，用来判断已问、已做、已表态；绝不复述。
- 把 `<?STATE>` 从“可自由大白话”改成固定三字段 schema：`关切=...；动作=...；表态=...`。三个字段每轮都出现、顺序固定；只要问了/做了/表态了，就不能全写“无”。

### P1 · 奥莱三大顾虑缺“消耗机制”，被当成任务清单反复续写

【现象】
- 正品 / 正价 / 奥莱差异：记录 357 第 3 轮问“奥莱跟专柜有啥区别/质量一样吗”，第 8 轮又问“奥莱的款跟专柜正价那种，皮子摸着有啥区别不”。记录 356 第 11 轮问“跟专柜一样吗/奥莱专门款吗”，第 12 轮追“小票或凭证”是合理追问，但后续应翻篇。
- 搭配是否合适：记录 359 第 1、3、4、5 轮都在“焦糖色/正式/周末/衣柜搭不搭”里打转。记录 355 第 2、3、4、5 轮也连续围绕颜色搭配和浅色耐用。
- 耐用 / 打理：记录 355 第 4、5 轮“划痕 / 抗造 / 磨花”，记录 356 第 7、8、9 轮“软皮塌 / 装满重 / 背久塌”，都是同类担忧的换皮复问。

【根因】
- “正品、划算、搭配”在画像里写得很强，但没有规定每类顾虑“导购答到什么质量才算消耗”“什么情况下可以追一句”“不满意时怎么表现而不是重问”。

【落点】
角色扮演链 → 几条守住的线 → 同担忧不换皮 / 奥莱三大顾虑消耗规则。

【改法】
- 正品/渠道安心：以答复质量消耗。导购给出正规门店、小票凭证、售后承接、渠道差异中的实在依据才算基本接住；只空口说放心时可以追一次依据，之后不再换说法重复。
- 划算/价格：导购讲清折扣/活动/奥莱价，或说明为什么这个价对她值，才算基本接住；新增费用如换肩带可追一次具体金额；仍含糊就记心里，不能第三次追同一金额。
- 搭配/会不会闲置：试背照镜子，或导购结合她的衣服与真实场景讲到具体画面，才算基本接住；之后最多再确认一个真实场景，不能把通勤、周末、衣柜、大地色、比例都轮流问一遍。
- 耐用/打理：浅色脏、刮花、磨花、抗造、保养是一组；软皮塌、撑变形、背久没型是一组。导购接住后不换词复问。

### P1 · 普通顾客视角偶尔越界成“产品质检员”

【现象】
- 记录 354 第 3 轮 Bella 说“不对吧，我手里摸着明明是带细绒的，不是那种光面皮。绒面的哪能防油防水啊。”前半句基于触感合理，后半句开始像在判导购的专业参数。

【根因】
- prompt 写了“不懂皮料学名”，但没有给“可用体感质疑、不可用专业名词判定”的表达边界。模型遇到 SA 说错材质时，容易过度强硬地站到产品专家视角。

【落点】
角色扮演链 → 产品事实边界。

【改法】
- 增加一句：你可以用“我摸着像绒面的，怕不好打理”这种体感话质疑；不要说“不是分层小牛皮 / 你这个材质不对”这类专家判定。

### P2 · 旁白仍偏多，且会反复演同类动作

【现象】
- 记录 359 5 轮里几乎每轮都有旁白，且多次“拿起 / 摩挲 / 调肩带 / 比划”。记录 355 也多次用摸皮、打开、看他放东西等动作推进。

【根因】
- 最新 prompt 已写“整场两三个关键时刻”，但在状态标签和旁白机制之间没有形成“动作已发生就别为了推进再演”的硬绑定。

【落点】
角色扮演链 → 旁白机制 / 状态机制。

【改法】
- `<?STATE>` 里把动作稳定记入 `动作=...`；后文明确“动作一旦进了 `{rolePlayState}`，后续只承认它已发生，不再用同类旁白补戏”。

## 本次 prompt 调整方向

本次不建议继续简单堆“不要重复问”。要改四个结构点：

1. `{rolePlayState}` 从“禁说内容”升级为“内部只读历史清单”。
2. `<?STATE>` 改成固定三字段 schema，服务后端提取。
3. 给 Bella 三大奥莱顾虑加“答复质量 + 追问上限”的消耗规则，避免画像底色变成无限任务清单。
4. 产品事实边界补“体感质疑 vs 专家判定”的分界。
