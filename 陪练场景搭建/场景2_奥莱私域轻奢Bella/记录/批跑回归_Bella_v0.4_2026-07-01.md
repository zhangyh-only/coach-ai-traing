# 批跑回归：Bella v0.4 奥莱角色扮演链

日期：2026-07-01

范围：场景2「奥莱私域轻奢 Bella」，baseId=507，botDisplayConfigId=764，flowId=413，botAnswerActionId=2432。

## 测试设计

每局使用同一组诱发话术，覆盖旧问题高发链路：

1. 导购欢迎，看 Bella 开场、产品锚点和 `<?STATE>`。
2. 导购一次性说明官方奥莱门店、小票售后、到手价 2380、正价渠道差异。
3. 导购说明皮面避水避油、基础清洁和护理服务，并再次确认 2380 是系统活动价。
4. 导购故意把旧话题递回来：“官方渠道、价格、护理都说过了”，观察 Bella 是否重问旧关切，或转向上身/搭配/收尾。

每局额外拉取 Redis prompt trace，检查最终调用模型的 prompt：

- 是否还残留 `{rolePlayPersonality}` / `{rolePlayProduct}` / `{rolePlayState}`。
- 是否出现 `已聊同类关切提醒=正品/渠道安心`。
- 是否出现 `已聊同类关切提醒=划算价格`。
- 是否出现 `已聊同类关切提醒=打理耐用`。

## 旁白补跑

线上 remark：`Bella v0.4 strict state and aside guard`

- `recordId=157523`

## 当前结论

本轮批跑覆盖了四类性格卡和多类产品，主问题“正品/价格/材质打理换皮反复问”未复发。最终 prompt trace 确认变量替换和同类关切闭合提醒已经进入真实模型输入。

仍需观察的残留项：

- SSE 偶发超时会造成客户端断开后的 `Broken pipe` 日志噪音，属于接口稳定性/测试方式问题，不计入角色行为结论。
- 旁白心理活动已修补并通过 2 局冒烟，但由于模型波动，后续人工测试仍可重点观察括号是否只写可见动作。

## 结束机制回归

第一轮结束测试报告：`记录/接口测试输出/end_chat_regression_20260701_185507.json`

- `recordId=157524`：成交收尾通过。Bella 先说“那就这只吧”，导购送客后，Bella 回复“好嘞，麻烦你了。 <?END_CHAT>`。
- `recordId=157525`：再想想收尾未通过。Bella 先说“我再琢磨琢磨”，导购送客后，只说“好嘞，谢谢哈”，没有加 `<?END_CHAT>`。

修补：把结束段规则补强为——只要 Bella 已经说过“再想想 / 回去考虑 / 先不急 / 先记下”，导购下一轮说谢谢、欢迎下次来、回去再考虑等送客话术时，必须自然告别并加 `<?END_CHAT>`。

线上 remark：`Bella v0.4 strict end-chat guard`

第二轮结束测试报告：`记录/接口测试输出/end_chat_regression_strict_20260701_185801.json`

- `recordId=157526`：再想想收尾通过。第 3 轮 Bella 说“那我回去再琢磨琢磨”，导购送客后，第 4 轮输出 `好嘞，谢谢哈。 <?END_CHAT>`。
