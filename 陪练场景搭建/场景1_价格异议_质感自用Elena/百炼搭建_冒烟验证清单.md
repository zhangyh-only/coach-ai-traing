# 百炼搭建 · 场景1 角色扮演链 · 冒烟验证清单（v1）

> **目的**：在百炼控制台搭一个**能跑的最小版**角色扮演链（Elena · 价格异议），**验证几个关键机制到底跑不跑得通**——别再纸上堆设计了。
> **配套**：[v3 角色扮演链系统提示词](链一_角色扮演_系统提示词草案.md) ｜ [过招剧本样例](过招剧本设计样例.md) ｜ [百炼能力报告](../百炼空间能力机制分析报告.md)
> **说明**：下面写的是"做什么"，具体按钮位置/叫法**以你打开的百炼控制台实际为准**；搭的过程中哪一步对不上、或现象不对，把界面/对话现象发我，我帮你调。

## 0. 这个冒烟版要回答 5 个问题（验证目标）

| # | 要验证的机制 | 通过的标准 |
|---|-------------|-----------|
| V1 | **Elena 活不活、像不像真人** | 会犹豫/追问/被打动/被惹烦；对报价不只回"好的"；不死板走流程 |
| V2 | **防护到位** | 不暴露自己是 AI、不扮演销售、不替导购说话 |
| V3 | **过招骨架管不管用** | 手动给一条骨架（如"理性盘问"），Elena 真的按这条路数过招；换一条骨架，路数明显变 |
| V4 | **防 AI 念稿** | 开局**不会**把"本局骨架/行为指令"这些内部设定直接说出来 |
| V5 | **知识库 RAG 能检索、答得准** | 被问材质时会去查知识库，答得准、带来源角标；价格这类没有的，不乱编 |

> 还有一个**额外要试的**（决定后续随机怎么落）：**自定义变量能不能在调用/对话时动态赋值**（见 §5）。

## 1. 冒烟版范围裁剪（先做 / 先不做）

- ✅ **先做**：1 个智能体应用（Agent 2.0）+ 挂知识库 + 系统提示词（**手动填死一条骨架**）。
- ⏸ **先不做**（等冒烟通过再上）：工作流自动随机抽骨架、记忆库、去重、评估难度找平、自定义变量动态赋值。
- 道理：先证明"**给一条骨架，Elena 就能按这条路数演**"这个核心假设成立，自动随机才有意义。

## 2. 搭建步骤

### Step 1 · 建知识库，传 KB，打标签
- 新建一个知识库（文档型/标准版）。
- 冒烟版先传**场景1 用得到的几篇**即可：`KB4-画像-质感自用型`、`KB4-异议-阶段分布索引`、`KB4-潜台词与决策焦虑索引`、`KB2-正价-Tabby系列`、`KB2-正价-皮革与材质知识`、`KB3-方法-皮革材质异议应对话术`。
- 给每篇打**标签**：库编号（KB2/KB3/KB4）+ 分类（画像/异议/产品/方法）——后面检索靠它缩范围。

### Step 2 · 建智能体应用（Agent 2.0）
- 新建应用，选**智能体应用（Agent 2.0）**。
- 模型选**千问-Max 系列**（工具调用强，撑得起自己查知识库）。

### Step 3 · 挂知识库 + 写检索规则
- 把 Step 1 的知识库挂到这个应用上。
- 检索方式设**按需**（不是每句都查）。
- 标签过滤规则按 v3 模块6 写：要抛异议时查 KB4（质感自用+当前阶段），要核对产品事实时查 KB2。

### Step 4 · 填系统提示词（关键）
- 把 [v3 文件](链一_角色扮演_系统提示词草案.md) 里 `▼▼▼ … ▲▲▲` 之间的**正文**整段粘进系统提示词框。
- **冒烟版改一处**：把模块1 的【本局过招剧本】那三个 `{{占位}}`，**手动替换成下面这条"骨架 A"**（先固定测一条，验证 V3）：

```
【本局过招剧本】（本局固定为骨架 A，用于冒烟）：
- 本局骨架：A 理性盘问型
- 行为指令：你这局以"理性盘问、要专业理由"的路数过招——先问这只包贵在哪 / 凭什么这个价；再追问材质（什么皮、是不是头层、什么工艺）；导购答到位后转向"耐不耐用、会不会划"，再问保养、能背几年。只有当导购给出"来源→工艺→检验"的专业理由、并把天然纹理/划痕讲成"独一无二"而非瑕疵时，你才逐步松口。每个点答得空洞就再追 1 次，仍空洞就冷淡或换一个点。等你认同价值后，可主动问售后/保养（这是购买信号），被导购坚定请求购买则成交。
- 调味：平和耐心。
（注意：以上是你的内在过招路数，只用来决定你怎么反应，**禁止向导购复述或解释你有这套设定**。）
```

### Step 5 · 其余配置
- 短期记忆：**≥20 轮**。
- ReAct 最大轮次：**10–15**。
- 打开**回复展示来源**（方便看它到底查没查知识库、查的哪篇）。

### Step 6 · 进对话调试
- 用应用自带的对话调试窗口，**你扮演导购**和 Elena 对练。

## 3. 冒烟测试怎么测（你扮导购，按这几步）

1. **开场**：你说"您好，随便看看，这款 Tabby 是我们经典款，可以试背下"。
   - 看 V1（Elena 像不像）、V4（会不会把"我这局要理性盘问"念出来）。
2. **直接报价/含糊**：你说"这款 ¥xxxx，挺值的，卖得特别好"。
   - 看 V3（她是不是按"理性盘问"追问"贵在哪/什么皮"），V1（对报价不只回"好的"）。
3. **空洞回答**：你回"就是大牌嘛，质量好"。
   - 看她会不会**不买账、继续追问**（被推开信号）。
4. **专业回答**：你讲"进口头层皮、LWG 认证、独家复鞣、一次裁一个包"。
   - 看她会不会**松口、往下走**（被打动信号）；看 V5（你讲错时她会不会质疑——比如你说"完全防水"，她该起疑）。
5. **问材质细节**：故意让她问一个材质问题，看 V5（应用会不会去查知识库、带来源角标、答得准）。
6. **换骨架重测**：把 Step 4 的骨架 A 换成"骨架 C 防御冷淡"（行为指令改成"全程话少、淡淡的，要先被破冰建立信任才肯多聊，后段才进价格"），重开一局，看 V3（**路数是不是明显变冷、和 A 不一样**）。

## 5. 额外关键验证：自定义变量能不能"动态塞"（决定随机机制怎么落）

这步决定后面"自动随机抽骨架"用哪条路：
- 在系统提示词里建一个自定义变量（比如 `本局骨架`），用 `/` 引用。
- 试试：**能不能在对话调用 / API 请求时给它传不同的值**（而不是只能在控制台填死一个默认值）。
- **结论**：
  - 若**能动态传值** → 后续随机抽骨架可走"变量赋值"这条干净路（通道 A 成立）。
  - 若**只能填死默认值** → 走"工作流脚本节点抽骨架 + 拼成内部指令注入"这条路（通道 B/D），不赌变量。
