## 概述

本文档记录退换货售后单从撤回、寄回、入库到退款确认的系统处理流程。它用于陪练中回答顾客关于退换货状态、寄回信息、退款节点和门店处理边界的问题；它不是完整退换政策文本。

## 核心内容

### 1. 售后申请撤回

| 节点 | 课程事实 |
|------|----------|
| 可撤回条件 | 退换货顾客未邮寄退回商品前，可根据实际情况撤销售后。 |
| 操作结果 | 确认撤销后关闭售后单。 |
| 顾客说明方向 | “如果您还没有寄回，我们可以根据实际情况协助撤销售后申请。” |

### 2. 售后处理流程

| 步骤 | 课程事实 |
|------|----------|
| 店长审批 | 店长审批通过售后申请后，系统自动创建售后单。 |
| 找到售后单 | 门店售后在系统中找到对应售后单。 |
| 返回地址 | 选择返回门店时显示门店地址；选择返回仓库时显示仓库收货地址。 |
| 返回仓库建议 | 为减少质检问题，自营店申请退货时建议优先退回仓库。 |
| 顾客寄回 | 店员把退货地址转给顾客，并沟通寄回退货商品。 |
| 填写物流 | 顾客寄回后，将物流公司和单号填写到售后单；列表没有对应公司时，可选择“其他”自行填写。 |
| 待退货 | 售后单进入待退货状态，等待退货商品寄回。 |

### 3. 退货入库处理

| 步骤 | 课程事实 |
|------|----------|
| 查找退货单 | 商品回到门店后，可切换到退货履约菜单，通过扫描退货包裹快递单号查找退货单。 |
| 包裹核验 | 点击退货入库后，通过退货包裹快递单号条码再次核验包裹是否与当前退货单一致。 |
| 商品验收 | 包裹核验一致后，打开包裹逐一检查退货商品，并扫码商品条码确认验收。 |
| 标签缺失 | 若商品标签缺失，需使用自定义条码完成退货。 |
| SA 检查 | SA 检查时尽量拍摄视频确认无误再点击下一步。 |
| 入库完成 | 退货商品验收合格后，点击确认入库完成退货。 |

### 4. 退款与换转退

| 场景 | 课程事实 |
|------|----------|
| 自收银门店 | 退货入库完成后，门店售后单将自动退款。 |
| 第三方收银门店 | 第三方 POS 退款后，Portal 上对应售后单需手动确认完成退款。 |
| 第三方收银含义 | 指非公司财务账号自收银以外的第三方收银，如联营商场。 |
| 换转退 | 申请换货后，在退回商品已入库、换货商品发货前，可根据顾客需求取消换货并申请退款。 |
| 店长审批 | 换转退需通知店长审批；通过后进入退款。 |

### 5. 导购可说 / 不宜说边界

| 顾客问题 | 可说方向 | 不宜说 |
|----------|----------|--------|
| “我还没寄回，能取消吗？” | “如果还没有寄回，可以根据实际情况协助撤销售后申请。” | “任何时候都能取消。” |
| “退回哪里？” | “系统会根据售后申请显示返回门店或返回仓库地址。” | “都退回门店就行。” |
| “退款什么时候到？” | “退货入库和门店收银类型会影响后续确认节点。” | “入库后一定立刻到账。” |
| “换货还能改成退款吗？” | “在退回商品已入库、换货商品发货前，可根据需求申请取消换货并退款，需店长审批。” | “随时可以换转退。” |

## 待确认事项

| 待确认项 | 影响 |
|----------|------|
| 课程未提供退换时限、商品状态要求、不可退换品类 | 陪练不能把本文档当完整政策。 |
| 不同渠道订单的售后入口和例外情况 | 需后续补充渠道政策。 |
| 退款到账时间 | 课程只描述系统节点，不说明到账时长。 |

## 在陪练系统中的应用

### 陪练对话引用

- 顾客问退换货、寄回地址、撤销售后、换货改退款、退款确认时引用本文档。
- AI 顾客可表现为担心送礼不合适、线上买到店处理、退回后不知道多久退款等。

### 对话评估引用

| 评估点 | 合格表现 |
|--------|----------|
| 流程理解 | 能区分撤回、售后处理、退货入库、退款确认。 |
| 渠道边界 | 能区分返回门店/返回仓库、自收银/第三方收银。 |
| 风险提示 | 不编造退换期限、到账时间和适用条件。 |
| 顾客安抚 | 能把流程说清楚，同时给顾客明确下一步。 |
