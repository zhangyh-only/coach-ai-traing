## 一、业务摸底结果（Coach 基本盘）

| 维度 | Coach 实际情况 | 对架构的影响 |
|------|---------------|-------------|
| 行业 / 监管 | 轻奢 / 时尚零售，**非强监管** | **不单设金融监管型 KB6**；交易、服务、退换、优惠边界归入 KB1 |
| 渠道形态 | 三渠道：**线下正价店 / 品牌奥莱 / 微信私域（小程序）** | 渠道是 KB3 顶层拆分轴 |
| 角色称谓 | **导购（学员）— 顾客**；语气贴近 GenZ、带朋友感，非"端着"的官方客服腔 | 写入 KB1 称谓规范 |
| 产品体系 | 轻奢包袋为核心：正价主推经典款、奥莱爆款、皮质、小皮具 / 鞋履 / 男装 | KB2 按品类与渠道拆 |
| **销售流程框架** | **线下正价店 6 阶段流程**：迎宾破冰 → 需求诊断 → 触感体验 → 价值传递 → 主动成交 → 售后维系；另有私域快速成交链路（变体）+ TA Card 方法论 | **决定 KB3**（v1.2 核心修正） |
| 销售方法论 | TA Card 技巧（识别购买信号 / 异议处理 / 全渠道销售 / 价格三明治）；流程中隐含假设成交法、以退为进等 | KB3 方法论独立成篇，跨阶段引用 |
| 顾客分类 | **4 行为类型**：质感自用型 / 性价比入门型 / 礼品选购型 / 都市悦己型（渠道作为属性，可与渠道画像资料合并补人设） | **本期核心**，KB4 中心 |
| 品牌理念 | **Expressive Luxury**；销售核心叙事 = 经典长期主义 / 质感投资 / 正品保障 | 写入 KB1 + KB3 价值传递阶段 |
| 评估资料 | 暂无历史陪练评估积累，但课程中的训练目标、方法步骤、Do/Don't、测验题、场景要求均可沉淀为评估点 | KB5 可先基于课程事实生成评估能力与检查点，后续再用历史样本校准权重 |
| 本期目标 | **陪练应用层**，以 AI 顾客画像真实化为第一优先级 | 采集 / 生成 deferred |

> **与太保的关系（v1.3 修正后）**
> 1. **Coach 有线性销售流程框架**（线下正价店 6 阶段），与太保"六程"**同构**——KB3 复用度比 v1.1 预想的更高。区别仅在于：Coach 无"续保"概念，无 T-N 时间轴；场景由"渠道 + 阶段"定义，而非续保倒计时。
> 2. **非强监管** → 取消金融监管型 KB6；沟通禁忌 / 话术红线并入 KB1、KB3，退换货 / 护理 / 刻字 / 优惠等交易与服务边界并入 KB1。
> 3. 其余方法论（下游引用驱动分库、画像 / 异议格式、KB5 功能分层、命名规范、单库+标签）直接复用。

### 1.1 资料来源优先级（v1.4 硬规则）

Coach 的知识库不是从场景需求文件里"推"出来的，而是从 Coach 已有课程资料里"抽"出来的。不同资料的职责如下：

| 资料类型 | 在本项目中的定位 | 可写入 KB 的内容 | 使用边界 |
|----------|------------------|------------------|----------|
| **课程资料** | **事实主来源** | 品牌定位、产品知识、材质/皮革、服务规则、退换流程、TA Card 方法论、GenZ 沟通原则、话术原文、训练重点 | KB 正文优先引用课程资料；课程未覆盖的事实不得用场景需求脑补 |
| 新场景要求 / 业务老师整理资料 | 场景需求与分类锚点 | 6 阶段流程骨架、4 行为类型、异议本质/潜台词、顾客画像希望呈现的方向 | 用来决定"怎么拆文档、哪些内容优先生成、角色要演成什么样"，不替代课程事实 |
| China Persona / 角色图 | 渠道人设补充 | 年龄、城市、职业、收入、渠道偏好、客单价、社交依赖、退货倾向等画像字段 | 只能补充 KB4 人设和渠道属性；不能作为 KB2 产品或 KB3 方法论事实来源 |

执行顺序也相应调整为：**先用内容分析提示词处理课程资料，产出结构化中间稿；再按 V1 文档清单生成 KB；最后用新场景要求/画像资料校准 KB4 和 KB3 的场景适配。**

**KB4 特别说明**：KB4 不等待专门画像资料。课程中出现的购买动机、适合人群、顾客异议、潜台词、决策焦虑、社媒偏好、服务焦虑、被哪类话术推开、对什么表达有正反馈，均可直接沉淀为 KB4 画像、异议索引或潜台词索引。业务老师资料、China Persona 和角色图用于补充分类、人设和渠道属性，不是生成 KB4 的前置条件。

## 二、设计思想：本期由"陪练对话"这个下游驱动

完整方法论有四个下游消费者，**本期采集与生成暂不展开，架构由"陪练对话"主导、"对话评估"共同支撑**：

| 下游消费者 | 本期状态 | 它向知识库要什么 |
|-----------|---------|-----------------|
| ① 对话采集 | 🔲 暂不纳入 | （后续）识别产品 / 渠道 / 阶段 / 顾客类型并匹配补全 |
| ② 变量生成 | 🔲 暂不纳入 | （后续）生成场景配置变量 |
| ③ **陪练对话** | ✅ **本期主线** | AI 扮演顾客：怎么想、怎么开口、被什么推开；就产品 / 正品 / 价格做真实回应 |
| ④ 对话评估 | ✅ 本期同步沉淀 | 判断学员是否走对流程阶段、用对技巧、是否踩 GenZ 沟通禁忌；把课程训练目标转为可观察检查点 |

本期优先级：

- **热路径 = KB4 顾客类型 + KB2 产品**：顾客类型决定"她怎么想、怎么说、被什么推开"；产品库给 AI 提供 FAQ 与回应边界。
- **KB3 流程与方法论 + KB1 背景**主要服务评估侧（学员是否走对 6 阶段、用对技巧），并为 AI 顾客提供场景边界。
- **画像真实度 > 知识覆盖度**：第一优先级是让 AI 顾客"演得像"。

> 继承的核心原则：粒度由"是否被独立引用"决定；**KB 文档是被检索、被提炼的参考材料，不是被整段复制的内容块**；异议写成"关切主题 + 心理 / 行为倾向（潜台词）"，绝不写完整剧本台词。

## 三、分库总览（五库 · 取消 KB6 · KB5 同步沉淀）

| 库 | 定位 | 企业特异度 | 拆分粒度策略 | 优先级 |
|----|------|-----------|-------------|--------|
| **KB1 品牌与渠道背景** | 品牌定位、三渠道特点、增值服务、交易与服务边界、角色与 GenZ 沟通规范 | 框架通用·内容特异 | 按主题拆；增值服务与交易/服务边界必须独立成篇（顾客会独立问到） | P1 |
| **KB2 产品知识** | 产品 / 品类详细信息、组合逻辑、奥莱与正价差异 | 内容特异 | 一个产品 / 品类一篇 + 体系总览 + 奥莱与正价对比 | P1 |
| **KB3 销售流程与方法论** | **线下正价店 6 阶段流程** + 私域快速成交 + TA Card 方法论 | **强企业特异** | **一个阶段一篇**（6 阶段）+ 流程总览 + 私域快速成交链路 + 方法论独立成篇（跨阶段引用） | P1（升档） |
| **KB4 顾客画像与异议应对** | 4 行为类型画像 + 异议总表 + 潜台词索引 | 框架通用·内容特异 | **一个行为类型一篇**（质感自用 / 性价比入门 / 礼品选购 / 都市悦己）+ 异议分类总表 + 潜台词 / 决策焦虑索引 | **P0（本期核心）** |
| **KB5 陪练评估框架** | 按功能分层的评估能力库 | 框架通用·内容特异 | 按功能分层（设计原则 / 维度参考库 / 评分检查点 / 场景匹配）；先基于课程事实沉淀，后续用历史样本校准权重 | P1 |
| ~~KB6 合规规范~~ | 零售非强监管，**取消金融监管型合规库**；沟通禁忌 / 话术红线并入 KB1、KB3，交易与服务边界并入 KB1 | — | — | — |

**销售流程框架（KB3 骨架，Coach 的"六程"）**：迎宾破冰（建立连接）→ 需求诊断（场景锚定）→ 触感体验（感官引导）→ 价值传递（故事加持）→ 主动成交（促成与连带）→ 售后维系（长期经营）。陪练场景可**按阶段选取**（如只练"触感体验"或"主动成交"）。

**顾客分类框架（KB4 主轴）**：4 行为类型 —— 质感自用型 / 性价比入门型 / 礼品选购型 / 都市悦己型。渠道（线下正价 / 私域 / 奥莱）作为画像属性字段，不单独分类。课程资料可直接提供购买动机、异议、决策焦虑和角色反应；业务老师初版、China Persona / 画像图用于合并补全分类和人设。

**A/B 渠道画像定位**：画像 A（线下正价轻奢包袋追求者）和画像 B（奥莱私域轻奢尝鲜族）不是 KB4 的并列主分类，而是"渠道 + 人设"补充资料。生成 KB4 时，先按 4 行为类型建文档，再把 A/B 中的年龄、城市、收入、渠道偏好、客单价、退货倾向等字段作为画像属性补入；无法确定映射关系时标注"【待人工确认】"，不强行归并。

> 拆分粒度判据："这块知识会不会被某个下游环节独立检索"。本架构只枚举 V1 首批建议文档清单，作为落地优先级；最终全集仍由后续资料处理和业务补充逐步扩展。

## 五、单文档格式规范（按知识类型）

给"章节骨架"，完整可填模板见配套《Coach 知识库文档生成 Prompt 模板》（prompt 模板需同步更新 KB3/KB4 两类）。

### 5.1 通用结构（所有文档）
元信息区块 → 概述 → 核心内容 → 在陪练系统中的应用 → 关联文档。

### 5.2 顾客画像类（KB4）—— 本期重点，含"角色扮演行为指引"
概述 → 基础人设（年龄 / 城市 / 职业 / 收入 / 社交圈）→ **渠道属性**（线下正价 / 私域 / 奥莱）→ 消费行为画像 → 心理动机与决策逻辑 → **角色扮演行为指引【本库特有】**（开口语气 / 会主动表达的关切 / 被什么话术推开 / 对什么有正反馈）→ 典型异议（**关切主题 + 潜台词/心理倾向**，口语化短句，**不写台词**）→ 异议应对方向 → 适配阶段 / 场景 → 应用 → 关联文档。

> 潜台词字段是 v1.2 强化项：业务老师初版的"顾客潜台词"（如"不确定值不值这个价"）直接喂"角色扮演行为指引"，让 AI 演出"没说出口的那层"。

### 5.3 产品类（KB2）
概述 → 产品基本信息表（全称 / 品类 / 渠道 / 系列 / 价位 / 皮质 / 容量 / 核心卖点）→ 详细介绍（设计灵感与结构 / 核心卖点 / 与相关产品对比 / FAQ / 常见异议及应对方向）→ 应用 → 关联文档。

### 5.4 销售流程 / 场景类（KB3）—— v1.3 修订
适用 6 阶段流程文档与私域快速成交链路文档（同结构）：
概述 → 阶段 / 场景定义（在 6 阶段流程中的位置或所属渠道 / 核心目标 / 销售强度）→ 客户状态描述 → 标准流程（按步骤）→ **该阶段典型异议及应对**（承接异议"按环节分布"）→ 关键动作与话术参考 → 培训重点环节建议 → 场景描述模板 → 应用 → 关联文档。

### 5.5 销售方法论类（KB3）
概述 → 方法论定位与适用场景 → 核心要点 / 步骤拆解（如 TA Card 各字母含义；GenZ 沟通禁忌"不要做的事+为什么推开"）→ 话术参考 → 常见误用 → 应用 → 关联文档。

### 5.6 背景类（KB1）/ 评估类（KB5）
KB1 沿用通用结构，但"增值服务体系"和"交易与服务边界"必须写清：权益内容、适用条件、顾客常问触发点、导购可说 / 不宜说的边界、评估时可检查的知识点。KB5 沿用太保"按功能分层"思路，但本项目不等待历史评估样本：课程中的训练目标、方法步骤、典型误用、Do/Don't、测验题、服务边界和产品介绍要求，都应先转为可观察的评估检查点；评分权重和细分档位可后续校准。

## 九、范围与边界

- **本期**：陪练应用层（陪练对话主线 + 评估辅助）；采集 / 生成为后续。
- **库**：本期 KB1–KB5；KB6 取消。
- **本期已就绪素材**：课程资料 + 业务老师初版（6 阶段流程 / 4 顾客类型 / 三视角异议）+ China Persona / 角色图。课程资料是事实主来源，业务老师资料是场景结构锚点。
- **待补素材**：私域 / 奥莱渠道流程的正式课程依据；4 类客户与课程内容的映射确认；服务权益、优惠、退换等规则的最新口径。
- **扩展原则**：内容扩充而非结构重构——新增阶段 / 类型 / 产品 = 新增文档，不动五库骨架。
