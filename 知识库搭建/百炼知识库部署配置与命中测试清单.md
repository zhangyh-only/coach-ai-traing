# Coach 陪练知识库 · 百炼部署配置与命中测试清单

> **定位**：知识库搭建阶段的**最后一环**——把已搭好的 46 篇 KB（md 文档）**上线到阿里云百炼**，成为三链可检索的知识库。前面已有「[架构设计 v2](Coach陪练知识库架构设计_v2.md) → [具体 KB 内容](具体KB内容/) → [业务反馈台账](知识库反馈/Coach知识库v2变更追溯台账.md)」，本文档补齐「**怎么建库、怎么切、怎么验**」。
> **归属**：属**知识库层**，不随场景变——任何陪练场景都用这同一个库、同一套切分检索配置。知识库迭代（加 KB / 改切分 / 重传）才会动它；场景迭代不会。
> **关联**：平台能力依据见 [百炼空间能力机制分析报告 §1.4 知识库 RAG 精调](../陪练场景搭建/参考资料/百炼空间能力机制分析报告.md)；单库+双标签约束见 [总体设计台账 §1](../陪练场景搭建/0_项目总览.md)。
> **状态图例**：✅ 直接照填 ／ 🔶 建议值（命中测试后可调）／ 🔧 需控制台实操验证

## 0. 两个前置认知（决定怎么填）

**① 建「一个」库，不是六个库。**
已锁定**单容器 + 双标签**（KB编号 + 分类）。46 篇全传进**同一个**知识库，靠标签区分 KB1–KB6 / 内容类别，三链检索时用标签限定范围。好处：维护一处、符合 RAG 元数据过滤最佳实践、角色扮演链可用标签**排除 KB5**（不让 AI 顾客"知道自己怎么被打分"）。

**② 这个三步创建表单只是"地基"，两件事不在表单里配：**
- **双标签**：在第 2 步「选择数据」上传文档时给每篇打 tag（或建库后在文档管理里维护）。
- **分链召回数**：表单里设的是知识库**默认**召回参数；三链是**独立 agent，各自在 agent 侧覆盖**召回数与标签范围（评估链召得多、角色扮演链召得少）。详见 §4。

## 1. KB 体量事实（数值依据）

| 事实 | 数字 | 影响哪个配置 |
|---|---|---|
| 单篇普遍很大 | 最长 KB4-异议主表 **≈3.3 万字符**（约 1.6 万 token）；KB3 多篇 2–2.5 万字符 | 分段长度必须**远大于默认 600** |
| 表格密度极高 | 全库 **约 999 行** Markdown 表格 | 切分要**防表格被拦腰截断** |
| 强标题结构（实测） | `##`=372 个、`###`=287 个、`####`仅 6 个；**知识单元跨 `##`/`###` 两级**（异议主表类在 `##`，退换货类在 `###`） | 适合**按标题切分**，标题级数=3 |
| 总量 | KB1=3 / KB2=11 / KB3=18 / KB4=5 / KB5=5 / KB6=4，共 **46 篇** | 初筛 TopK 不必压低 |

## 2. 配置照填卡

### 2.1 第 1 步 · 基础信息

| 项 | 推荐 | 理由 |
|---|---|---|
| 知识库类型 | **文档搜索** ✅ | md 文本混合索引；不是 NL2SQL 表（数据查询），不是图片/音视频 |
| 使用场景 | **基础文档问答** ✅ | 召回切片→模型用。不选「图文并茂」（无图）、「视觉理解」（非扫描富文本件）、**「极速问答」**（面向高度结构化简单文档，我们文档复杂、表格多，不符） |
| 知识库名称 | `Coach蔻驰销售陪练知识库`（≤20字） | — |
| 知识库描述 | 见下方可粘贴文本 | 百炼明示"描述用于指导智能体调用知识库"，**必须写清覆盖范围** |

**知识库描述（可直接粘贴）：**
> Coach 蔻驰门店销售陪练知识库，含品牌渠道与红线(KB1)、产品材质与FAQ(KB2)、销售流程与方法论(KB3)、顾客画像与异议应对(KB4)、陪练评估框架(KB5)、门店服务规范(KB6)六大类。供角色扮演/实时评估/整体评估三链按 KB编号+分类标签按需检索；产品价格材质等事实层内容为唯一事实源，无数据标【待补】不臆造。

### 2.2 第 2 步 · 选择数据（上传 + 打标签）

- **上传**：`知识库搭建/具体KB内容/` 下 46 篇 md 全量上传。
- **双标签草案**（最小可用，🔧 控制台确认打标方式）：

| 标签维度 | 取值 | 用途 |
|---|---|---|
| **标签A · KB编号** | `KB1` / `KB2` / `KB3` / `KB4` / `KB5` / `KB6` | **分链限定范围**（角色扮演链排除 KB5；评估链限定 KB5+KB3） |
| **标签B · 内容类别** | `品牌红线` / `产品FAQ` / `流程方法` / `画像异议` / `评估框架` / `服务规范` | 同链内进一步缩小检索面、提准降 token |

> 画像类型 / 阶段 / 产品 等更细的检索标签，待 agent **KB 检索规则**（总体设计 §3）定稿时再在文档级补充，不在本次建库一次到位。

### 2.3 第 3 步 · 索引设置

| 项 | 推荐值 | 理由 / KB 依据 |
|---|---|---|
| **切片方式** | **按标题切分**（主推）｜智能切分（兜底）🔶 | KB 按方法论写成强结构，知识单元落在 `##`/`###` 两级（一条异议/一个画像维度/一条评估标准/一段 SOP）；按标题切让**每片≈一个完整知识单元**、主题不混片、检索最准 |
| **└ 标题级数** | **3**（默认 1 必改！）🔶 | 决定切到第几级标题。全库实测 `##`=372 / `###`=287 / `####`=6；知识单元**跨 ##/### 两级**——退换货类"`##`核心内容 → `###`一二三知识单元"，异议主表类"`##`一二三视图直接挂表"。**切到 `###`（级数3）两类都能切开**；级数 1/2 会把多个知识单元挤成一大片被长度截，级数 4 无收益 |
| **最大分段长度** | **3000**（默认 600 远不够；初设 1000 仍偏小）🔶 | 一条异议/一张视图表/一段 SOP 常 >1000 token；3000 兜住绝大多数知识单元节。**特例**：异议主表"`##`一"是 45 条超大单表，3000 可能仍超 → 见命中测试 T4/T5，被截则该篇单独拉到 6000 或按异议编号预处理拆段 |
| Metadata 抽取 | **关**（默认） | 双标签靠**手动打 tag + 文档头部「文档编号/所属知识库」字段**，可控、过滤准；LLM 自动抽 metadata 不稳又费 token |
| Excel 表头拼装 | 无所谓 | 只对 .xlsx 生效，对 md 表格不起作用 |
| **多轮对话改写** | **开** ✅ | 陪练是多轮对话，query 常带指代（"那这款呢"），改写补全后再检索能提召回 |
| 向量模型 | **text-embedding-v4** ✅ | 通义最新通用文本向量，中文 + 大量专业术语合适 |
| **排序模型** | **qwen3-rerank(hybrid)** | KB 有海量专名/编号（TABBY、55387、FABS、BFA、O01–O45、Pebble Leather）；hybrid 融合关键词特征，避免纯语义对专名漏召 |
| 排序模型模式 | **问答模式** ✅ | RAG 问答（agent 提问→召回应对），按 Query-答案匹配排序最贴合 |
| 初步向量/关键词 TopK | **50 / 50**（默认） | rerank 前初筛池，够大；交给 rerank 精排 |
| 相似度阈值 | **0.20 起步** 🔶 | 偏宽松、先保召全（异议/画像要召全），靠 rerank+最终数控质量；噪声多再升 0.3 |
| **最终召回最大数量** | 库默认 **8** 🔶；分链在 agent 侧覆盖（§4） | 折中值；角色扮演链调低、评估链调高 |
| 向量存储 | **平台存储** ✅ | 推荐、免费、异地容灾、免运维；ADB-PG 是自购自担运维，没必要 |

## 3. 分链召回参数（在各 agent 侧覆盖知识库默认）

> 知识库默认召回数是"一刀切"，三链需求不同，**在每个 agent 的知识库检索配置里分别覆盖**。

| 链 | 标签范围 | 最终召回数 | 调用方式 | 理由 |
|---|---|---|---|---|
| ① 角色扮演 | KB1/KB2/KB3/KB4/KB6（**排除 KB5**） | **5** | 按需 | 顾客别"知道太多"掉书袋；聚焦当前画像+相关异议 |
| ② 实时评估 | KB5 + KB3（当前阶段） | **10–15** | 必定 | 评最新一条销售消息需召全维度判断+红线 |
| ③ 整体评估 | KB5（四件套）+ KB3/KB4 | **15–20** | 必定 | 列举四维度全部指标，属"列举类"，百炼建议调到 20，否则漏维度 |

> 同一个 query，分别在**评估链范围**（含 KB5）和**角色扮演链范围**（排除 KB5）下各检索一次，预期**相反**——这同时验证了召全与隔离两件事。

| # | 检索 query | 评估链（含KB5）应：命中 + 召全 | 角色扮演链（排除KB5）应：**召不到** |
|---|---|---|---|
| T17 | 陪练评估四维度各占多少分权重 | 命中 KB5-评分-评分规范：专业性30 / 同理心25 / 沟通技巧25 / 流程意识20 | 召不到任何 KB5 切片（顾客拿不到打分逻辑） |
| T18 | 评分档位优秀良好待提升各多少分 | 命中三档区间表：专业性满分30→优秀25-30/良好20-24/待提升0-19 | 召不到 |
| T19 | 评估方案里哪些指标类型绝对不允许出现 | 命中 KB5-设计：时间类/语音技术类/机械话术类/表情动作类/假设结果类（5类召全） | 召不到 |

### 4.6 不通过 → 怎么调（对照表）

| 症状 | 调整动作 |
|---|---|
| 大表被拦腰截（T1–T6 字段缺尾部/表只剩半张） | 该篇回退「智能切分」／ 调大最大分段长度 ／ 预处理把超大表拆节 |
| 专名漏召（T7–T12 召不到或排序靠后） | 确认排序模型 = **qwen3-rerank(hybrid)**；初筛 TopK 提高；相似度阈值降低 |
| 列举不全（T13–T16 缺项） | 该 agent 最终召回数提到 **15–20**；分段长度调大让整表进一片 |
| 隔离失败（T17–T19 角色扮演链**召到了** KB5） | 检查该 agent 标签范围是否排除 KB5、KB5 文档是否打了 KB5 标签 |
| 多轮改写无效（T20–T21 轮2 召错） | 确认「多轮对话改写」已开启 |

## 5. 落地步骤

1. 控制台 → 知识库 → 创建知识库，按 §2.1 / §2.3 填三步表单。
2. 第 2 步上传 `具体KB内容/` 46 篇，按 §2.2 打双标签（🔧 确认打标 UI）。
3. 完成建库，等切片+向量化完成。
4. 进「命中测试」逐条跑 §4 清单，按结果回调切片/阈值/召回数。
5. 三链 agent 搭建时，按 §3 在各 agent 侧覆盖标签范围与召回数。
6. 结论与调参回写本文档「变更记录」。
