## 〇、本次修订背景（必须先看清的事实）

对业务老师替换进仓库的 44 个 KB 文件逐一核对后（用 word-diff / 剥离格式后字节比对）发现：

1. **她的文件改动约 95% 是"格式噪声"**：把文档在某编辑器/在线文档里过了一遍粘回来，破坏了几乎所有 Markdown 表格（表头错位、`>` 强调引用块被吞、对齐与末尾换行丢失）。**真正动手改内容的全库合计只有 3 处**（KB1 的 A.L.L. 一处措辞、KB2 连带补一段、KB2 颜色词典夹了一句审阅批注）。
2. **她写在反馈文档里的十几条意见，绝大多数没落到文件**——仍是"反馈意见"，需要我们正式执行。
3. **评估维度/权重/总分未被改动**（专业性30·同理心25·沟通技巧25·流程意识20=100，安全）。
4. 处理原则（已与用户确认）：**回滚格式噪声、恢复规范 Markdown，只保留 3 处真实有效改动中有价值的部分**；反馈意见由我们正式落地；缺失课程环节据《The Coach Experience》真实页面补录，**绝不臆造**。

## 二、四项关键架构变更详解

### 变更 1：新增 KB6 门店服务规范（澄清：不是恢复"金融合规库"）

- **为什么不矛盾**：v1.4 的 D1 否决的是"**金融监管型合规库**"（轻奢零售非强监管，确实不需要）。本次新建的 KB6 是"**门店服务规范库**"——性质是**服务操作 SOP**，不是合规。业务老师的论点成立：KB1 的维度是"品牌 + 渠道"，而"退换怎么办、护理怎么收费、能否刻字、COACH CREATE 怎么做"是**门店服务标准**，**不分渠道**，混在品牌渠道库里维度不纯。
- **KB6 收纳**：
  - `KB6-服务-交易与服务边界(退换货)`（从 KB1 迁入）
  - `KB6-服务-增值服务体系`（从 KB1 迁入）
  - `KB6-服务-COACH皮革服务`（**新建**，源 The Coach Experience p13：印字/烫印、五金维护、皮革保养与护理、皮革工艺加工）
  - `KB6-服务-COACH CREATE个性化定制`（**新建**，源 p14：Personalize / Customize / Accessorize / Make It To Order）
- **边界规则**：**"服务是什么、规则边界" → KB6（静态事实）**；**"销售时如何用服务打动顾客"（话术/动作） → KB3 流程**（如 BUILD 阶段调用 KB6 服务卖点）。

### 变更 2：CONNECT/BUILD/CLOSE/CONTINUE 四阶段确立为官方主框架

- **来源权威性**：四阶段是 Coach 官方课程《The Coach Experience》的顶层框架（p01、p05）；原"线下正价店 6 阶段"是业务老师/思创为陪练做的细化。二者是 **1:N 映射**，不冲突。
- **业务老师给定的映射**（已确认，可销项原文档"映射待补充"）：

  | 四阶段（官方） | 六阶段（细化映射） |
  |---|---|
  | **CONNECT 联系** | 迎宾破冰（建立连接）+ 需求诊断（场景锚定） |
  | **BUILD 打造** | 触感体验（感官引导）+ 价值传递（故事加持） |
  | **CLOSE 成交** | 主动成交（促成与连带） |
  | **CONTINUE 延续** | 售后维系（长期经营） |

- **落地方式（不重构 16 篇，只做对齐）**：
  1. `KB3-流程-线下正价店6阶段总览` → 升级为 `KB3-流程-COACH四阶段与六阶段总览`，去掉"正价店"限定（业务老师："与是否正价店无关"），表头改为"四阶段主框架 + 六阶段细化"，并标注**售后维系/CONTINUE 不适用于所有短链路场景**。
  2. `KB3-方法-TA Card销售技巧总览` → 把"四↔六映射【待补充】"用上表**销项**。
  3. 各阶段详解文档 → 元信息补"所属四阶段（CONNECT/BUILD/CLOSE/CONTINUE）"。
  4. `KB5-场景-场景类型×维度匹配` → 阶段轴改为"四阶段为主、六阶段为通俗映射"。

### 变更 3：GenZ 活人感沟通 KB1 → KB3 合并去重

- `KB1-沟通-GenZ活人感沟通规范与红线` 迁入 KB3，与现有 `KB3-方法-GenZ活人感沟通法则` 配套：**红线/禁忌**部分并入"对客沟通禁忌"段，**正向方法**（活人感/SBI/SBIR/冰山模型）保留，避免红线两处维护漂移。
- 补业务老师要求：在 GenZ 法则里补一句 **GenZ 常规定义（泛指的年龄群体）**，作为"何时调用本库"的触发判据。

### 变更 4：KB4 三个异议索引合并为"单主表 + 三视图"

- 现状：`类型分类总表`/`阶段分布索引`/`潜台词与决策焦虑索引` 三篇**行项是同一批异议**，只是分别挂在 类型/阶段/焦虑 三个维度上，**潜台词与应对方向互相重复**，业务老师"看不出区别和重点"成立。
- 合并方案：
  - 建 **`KB4-异议-异议主表`**：每条异议一行，列含 `关切主题 | 异议类型 | 出现阶段 | 关联行为类型 | 潜台词 | 应对/心理按摩方向 | 焦虑族 | 来源课程 | 置信度`。**唯一事实源**。
  - 三个原索引降级为**视图**（文档内"按类型看 / 按阶段看 / 按焦虑族看"三个透视章节，只给排序+跳回主表），不再各存一份潜台词与应对，根除重复与漂移。
  - "心理按摩方向总纲（8 主轴）""关联行为类型速查"作为主表附录保留（真正不重叠的提炼层）。
